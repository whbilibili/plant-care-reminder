## Product Decisions

- 登录注册不拆成两个独立页面。使用同一页面内的 `登录 / 注册` 双模式表单，方便邀请链接场景下未登录用户快速完成账号创建并继续加入家庭。
- 设置页的邀请码区同时提供两个复制动作：主按钮 `复制邀请链接`，次级快捷按钮 `复制邀请码`。前者用于直接分享给家人，后者用于口头告知、截图或只发送 6 位码。

## Visual Direction

- 页面底色：暖纸色 / 淡绿纸面，不使用认证后大面积渐变。
- 顶部结构：轻量导航行优先，避免独立白色横条；标题与当前对象信息避免重复。
- 内容层级：先用间距、分组和分割线，再使用边框，阴影最后使用。
- 卡片策略：不要卡片套卡片；列表优先是 grouped surface + row divider。
- 任务动作：完成按钮保持清晰、靠右、可单手点击。
- 图片策略：植物图片是识别锚点，不是装饰。

## Implementation Notes

建议下一轮按以下顺序实现：

1. 建立共享 `ScreenNav` / `ObjectSummaryBand` / `GroupedSurface` 三个布局基元。
2. 先改植物详情页顶部，验证「轻量原生导航」能解决白条贴片问题。
3. 再改待办页和植物看板，统一任务卡、植物卡、底部导航节奏。
4. 最后改表单与设置页，复用同一套顶部和 grouped surface 规则。

实现前需要把每张图拆成页面验收项，避免一次性大改造成视觉回归。
