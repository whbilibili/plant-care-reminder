# 2026-06-09 前端页面调整方案

**项目**：plant-care-reminder  
**基线来源**：
- [docs/design-docs/moodboard/README.md](./moodboard/README.md)
- [docs/design-docs/core-beliefs.md](./core-beliefs.md)
- [docs/DESIGN.md](../DESIGN.md)

## 1. 目标与本轮范围

本轮不是“换一套新视觉”，而是把现有实现拉回到最初设计稿的产品方向上。核心目标有三条：

1. 把“任务优先”重新做成主信息架构，而不是让说明性大标题占据首屏。
2. 把“植物图片是识别锚点”落实到植物列表和详情页，而不是出现空白大块或坏图体验。
3. 把“家庭协作”做成可操作的设置与加入体验，尤其解决邀请码复制失败、退出登录不显眼、中文语境不统一的问题。

本轮只覆盖移动端 PWA 主路径：

- 登录
- 首次资料填写
- 创建家庭 / 加入家庭
- 植物列表
- 植物详情
- 待办页
- 设置页

## 2. 问题归纳

### 2.1 当前实现的主要偏差

1. 文案层面仍混有英文，且部分表达更像国际化占位文本，不像中国家庭会自然使用的语言。
2. 邀请码复制依赖浏览器剪贴板能力，一旦失败就只给出错误提示，没有继续完成任务的备用路径。
3. 多个页面延续了“大 Hero + 长段说明 + 下方操作”的结构，导致首屏信息密度偏低，不符合高频家庭操作场景。
4. 植物图虽然支持上传，但列表页和详情页的图片位仍然不够“主角化”，在无图/坏图时也缺少有意设计的占位状态。
5. 设置页与待办页目前的信息层级偏平均，没有把“当前最常做的动作”压到最靠近手指的位置。

### 2.2 与原始设计稿的一致方向

根据 moodboard，应该回到以下方向：

- `01-task-first-dashboard.svg`：待办优先，第一屏先看到今天要做什么
- `02-botanical-card-system.svg`：卡片系统围绕植物识别和状态扫描展开
- `05-quiet-form-flow.svg`：表单应该安静、紧凑、低认知负担
- `06-family-sync-settings.svg`：设置页应突出共享关系、邀请码和家庭协作
- `08-photo-led-plant-list.svg`：植物列表应由照片驱动识别

## 3. 总体设计调整策略

### 3.1 信息架构

- 登录后默认主入口建议切到"待办"，而不是"植物"。
- 底部导航保留三项，但语义建议固定为：`待办 / 植物 / 设置`。
- 每个页面首屏只允许一个主要动作，不再让说明文字占据主要视觉权重。

### 3.2 视觉色彩与风格

> **重要差异说明**：`docs/DESIGN.md` 中定义的色彩 Token（蓝色系 `#2563EB` + 橙色 CTA `#F97316`）来自 `ui-ux-pro-max` 自动生成，尚未与 moodboard 对齐。moodboard.html 中实际使用的设计色系是"叶绿 + 雾蓝 + 暖白"。本轮改造以 moodboard 的视觉方向为准，需要同步更新 `DESIGN.md` 中的 Token。

#### 3.2.1 目标色彩系统（以 moodboard 为准）

```css
:root {
  /* 文字 */
  --color-ink: #16342f;           /* 主文字，深墨绿 */
  --color-muted: #637b71;         /* 次要文字/辅助说明 */

  /* 背景与表面 */
  --color-paper: #fbfcf7;         /* 页面底色，暖白 */
  --color-mist: #edf5f1;          /* 卡片/区块底色，薄荷雾 */
  --color-surface: #ffffff;       /* 浮层/Modal 表面 */

  /* 品牌与交互 */
  --color-leaf: #1f473d;          /* 主色，深叶绿 */
  --color-leaf-light: #467061;    /* 主色浅变体，用于 eyebrow/标签 */
  --color-gold: #f1c567;          /* CTA/强调色，暖金 */
  --color-gold-hover: #ddb04e;    /* CTA hover 态 */

  /* 边界与分割 */
  --color-line: #d8e4da;          /* 边框/分割线，浅灰绿 */

  /* 语义色 */
  --color-success: #2f855a;       /* 完成/成功 */
  --color-warning: #dd6b20;       /* 逾期/警告 */
  --color-error: #c53030;         /* 错误/危险 */

  /* 装饰性背景渐变（仅登录页等氛围页使用） */
  --gradient-botanical: linear-gradient(135deg, rgba(235, 246, 238, .95), rgba(246, 249, 239, .88));
  --gradient-accent: radial-gradient(circle at 82% 8%, rgba(181, 212, 220, .55), transparent 34%);
}
```

#### 3.2.2 色彩使用规则

- 主文字始终使用 `--color-ink`，不要用纯黑 `#000`。
- 次要/辅助文案使用 `--color-muted`。
- 页面背景统一 `--color-paper`，卡片使用 `--color-mist` 或 `--color-surface`。
- 主按钮填充色为 `--color-leaf`，文字白色；hover/active 态加深 10%。
- CTA 强调按钮（如邀请码复制、完成任务）使用 `--color-gold` 填充，文字用 `--color-ink`。
- 次按钮使用 `--color-leaf` 描边 + 透明底，不再使用蓝色系。
- 卡片边框统一 `1px solid var(--color-line)`，强调卡片可加深到 `#87af9a`。
- 装饰性渐变背景仅限登录页和空状态引导页，认证后的操作页面使用纯色底。

#### 3.2.3 字体规范

- 标题字体：`Lora`（衬线，提供仪式感）
- 正文字体：`Raleway`（无衬线，清晰可读）
- 中文回退：`"PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", sans-serif`
- 字号层级（移动端）：
  - 页面主标题：`24px`，`font-weight: 700`，行高 `1.2`
  - 卡片标题：`16px`，`font-weight: 600`，行高 `1.25`
  - 正文/说明：`14px`，`font-weight: 400`，行高 `1.5`
  - 辅助/标签：`12px`，`font-weight: 700`，行高 `1.3`，可用 uppercase/letter-spacing

#### 3.2.4 视觉节奏与密度

- 缩短首屏标题区高度，认证后页面的 Header 区总高度不超过 `120px`（含标题 + 一行说明 + 间距）。
- CTA 保持明确，但不要每页都出现过大的整幅色块按钮；主按钮宽度不超过 `calc(100% - 48px)`。
- 卡片内部改为"内容分组优先"，卡片内 padding 统一 `16px`，组间距 `12px`。
- 植物卡片封面图比例固定为 `16:10`（`aspect-ratio: 16/10`）。
- 底部导航高度 `56px`，图标 `24px`，标签 `10px`。

#### 3.2.5 阴影与层级

- 卡片阴影：`0 18px 42px rgba(36, 73, 63, .12)`（安静植物感，不要浓重工业阴影）
- 强调卡片：`0 22px 54px rgba(31, 71, 61, .2)`
- 浮层/Sheet：`0 -8px 32px rgba(36, 73, 63, .15)`
- 不使用 `box-shadow: 0 4px 6px rgba(0,0,0,.1)` 等通用灰色阴影

#### 3.2.6 圆角规范

- 卡片：`16px`
- 按钮：`12px`（中号）/ `999px`（胶囊/Pill）
- 输入框：`12px`
- Modal / Sheet：`20px`（顶部两角）
- 标签/Badge：`999px`

### 3.3 文案策略

- 全部主路径统一为简体中文。
- 少用"共享家庭空间/household board"这类偏产品说明的翻译腔。
- 优先使用中国家庭更自然的词：`家庭`、`家人`、`邀请码`、`植物看板`、`养护任务`、`你的称呼`。

## 4. 页面级详细调整方案

## 4.1 登录页 `/login`

### 现状问题

- 首屏说明文字偏长。
- 登录和注册切换区块占比过大。
- 整体更像营销引导，不像高频工具入口。

### 调整目标

- 让用户 3 秒内理解"输入邮箱密码即可进入家庭植物看板"。
- 缩短文案，压缩首屏高度，让输入框更早进入视野。

### 具体调整

- 主标题从大段品牌式语句调整为更直接的工具语义。
- 说明文案缩成 1 到 2 行，强调"登录后进入家庭植物看板"。
- 登录 / 注册切换保留，但减少外层胶囊高度和阴影强度。
- 表单区域与错误提示之间保留更紧凑的垂直节奏。

### 视觉规格

- 背景：使用 `--gradient-botanical` + `--gradient-accent` 组合，这是全 app 唯一使用渐变背景的页面。
- 标题区总高度上限 `160px`（含 logo/品牌图标 + 标题 + 一行说明），确保输入框在 375px 屏幕不需要滚动即可见。
- 登录/注册切换胶囊：高度 `36px`，圆角 `999px`，背景 `rgba(255,255,255,.76)`，边框 `1px solid var(--color-line)`。
- 输入框：高度 `48px`，圆角 `12px`，背景 `var(--color-surface)`，边框 `1px solid var(--color-line)`，focus 态边框色 `var(--color-leaf)`。
- 主登录按钮：高度 `48px`，宽度 `100%`，圆角 `12px`，背景 `var(--color-leaf)`，文字白色 `14px/600`。
- 表单卡片：背景 `rgba(255,255,255,.82)`，圆角 `16px`，阴影 `0 18px 42px rgba(36,73,63,.12)`，padding `24px`。

### 交互要求

- 登录失败时，错误提示紧贴表单，不把主按钮挤到屏幕外。
- 登录成功后，状态提示文案简短，直接进入下一步。

## 4.2 资料初始化页 `/onboarding/profile`

### 调整目标

- 让用户知道这是“家人如何称呼你”的设置，而不是抽象的 profile setup。

### 具体调整

- 标题用口语化中文：`家人该怎么称呼你？`
- 表单只保留一个输入项和一个提交按钮。
- 减少说明文案长度，把“名字会出现在任务记录中”的说明放成一行辅助文案。

## 4.3 家庭接入选择页 `/onboarding`

### 现状问题

- 入口卡片表达还不够“一个是新建，一个是加入”的直接决策。
- 页头仍偏介绍式，不够像流程分叉页。

### 调整目标

- 让用户一眼只做一个判断：`创建家庭` 或 `加入家庭`。

### 具体调整

- 两张卡片保持并列纵向排列，但内容压缩到：角标、标题、1 行描述、按钮。
- 去掉冗长的段落说明。
- `创建家庭` 作为主按钮，`加入家庭` 作为次按钮，但两者都需要足够明显。

## 4.4 创建家庭页 `/onboarding/create-family`

### 调整目标

- 把流程做成一个安静的单表单页。
- 成功态的重点是“邀请码”本身，而不是解释性文案。

### 具体调整

- 表单卡片中只保留：标题、家庭名称输入、主按钮。
- 成功态卡片上半部分直接放大邀请码，下方放一句“发给家人即可加入”。
- `进入植物看板` 按钮保留，但层级低于邀请码本身。

### 交互要求

- 输入合法时可直接提交，不增加额外确认。
- 成功后不自动跳走，优先让用户看到邀请码。

## 4.5 加入家庭页 `/onboarding/join-family`

### 调整目标

- 输入邀请码的流程要短、稳、无歧义。

### 具体调整

- 单输入框布局，输入框标题使用 `邀请码`。
- 输入提示明确写 `向家人索取 6 位邀请码`。
- 成功后进入中间态页，但中间态只保留一个简短标题和一句同步说明。

### 错误态要求

- 如果邀请码无效，错误文案直接说明“邀请码不正确或已失效”。
- 错误提示应出现在输入框附近，而不是页面底部。

## 4.6 待办页 `/todo`

### 现状问题

- 虽然已有任务分组，但首屏仍可能先看到大段说明而不是任务本身。
- 任务完成动作和任务信息有时竞争视觉优先级。

### 调整目标

- 待办页成为真正的"家庭首页"。

### 具体调整

- 首屏先看到今天要处理的任务分组，不再用过高的 Hero 区。
- 每张任务卡片只突出三件事：植物名、任务动作、完成按钮。
- 到期状态使用清晰的文案和轻量色块，不做大片装饰背景。
- 说明性文案压缩为组标题下的一句话或直接删除。

### 视觉规格

- 页面背景：`var(--color-paper)`，无渐变。
- 标题区：仅一行 `24px` 标题（如"今日待办"），不设 Hero；标题到第一张卡片的间距 `16px`。
- 分组标题：`12px` uppercase 标签样式，颜色 `var(--color-leaf-light)`，letter-spacing `0.08em`。
- 任务卡片：背景 `var(--color-surface)`，圆角 `16px`，padding `16px`，卡片间距 `12px`。
- 卡片内布局：左侧植物缩略图（`40px` 圆形），中间植物名 + 任务动作文案，右侧完成按钮。
- 完成按钮：`36px` 圆形，背景 `var(--color-gold)`，内含 checkmark icon `20px`。
- 逾期标记：使用 `var(--color-warning)` 文字 + `rgba(221,107,32,.08)` 背景色块，圆角 `999px`，padding `4px 8px`。
- 空状态：居中一句话 + 一个建议动作按钮，不做大面积插画。

### 交互要求

- 完成按钮在首屏内可直接点击，尽量靠近右下或拇指区。
- 完成后的反馈使用轻量状态变更（卡片淡出 + 短暂 `var(--color-success)` 勾选动画），不阻断浏览下一条任务。
- 列表滚动时底部导航保持可见，不被内容遮盖。

## 4.7 植物列表页 `/plants`

### 现状问题

- 当前页面顶部说明偏长。
- 植物卡虽然有图片区，但图片区没有成为最强识别元素。
- 无图/坏图时视觉结果接近“空白块”。

### 调整目标

- 植物列表页是“按照片识别植物”的看板，而不是文字目录。

### 具体调整

- 缩短页面标题区，只保留标题和新增植物按钮。
- 搜索栏保留，但压缩到更贴近列表的位置。
- 植物卡片结构改为：
  - 顶部：16:10 左右的封面图
  - 中部：植物名 + 所在位置
  - 下部：下一次养护 / 当前无任务
  - 操作：进入详情、编辑
- 无图状态使用带叶片/花盆语义的占位块，而不是纯空白背景。

### 视觉规格

- 页面背景：`var(--color-paper)`。
- 标题区：仅标题（`24px`）+ 右侧"添加植物"按钮，总高度 `48px`，下方紧接搜索栏。
- 搜索栏：高度 `40px`，圆角 `12px`，背景 `var(--color-mist)`，placeholder 色 `var(--color-muted)`。
- 植物卡片：
  - 封面图区域：`aspect-ratio: 16/10`，`object-fit: cover`，圆角顶部 `16px`。
  - 信息区 padding：`12px 16px`。
  - 植物名：`16px/600`，颜色 `var(--color-ink)`。
  - 位置标签：`12px`，颜色 `var(--color-muted)`。
  - 下次养护 Pill：`12px/700`，背景 `var(--color-mist)`，颜色 `var(--color-leaf-light)`，圆角 `999px`。
- 无图占位态：背景 `var(--color-mist)`，居中放一个 `48px` 的叶片线稿 icon（颜色 `var(--color-line)`），底部一行 `12px` 文案"轻触上传照片"。
- 卡片间距：`16px`（纵向列表布局）。

### 图片展示要求

- 有图时默认铺满裁切显示。
- 图失效时自动尝试刷新 URL。
- 仍失败时显示有意设计的占位态和简短提示。

## 4.8 植物详情页 `/plants/:plantId`

### 调整目标

- 详情页要同时承担“查看植物档案”和“管理养护任务”两个职责，但不能上下脱节。

### 具体调整

- 顶部 Hero 缩短，减少大面积说明文案。
- 先展示图片、植物名、位置和一句状态信息。
- 任务列表直接上移，避免用户先读很多静态信息才进入操作区。
- 编辑植物、添加任务等动作分组收纳，不与任务完成操作抢层级。

### 视觉规格

- 页面背景：`var(--color-paper)`。
- Hero 图片区：`aspect-ratio: 4/3`，`width: 100%`，`object-fit: cover`，圆角底部 `24px`（顶部与屏幕齐平）。无图占位态与植物列表页一致：背景 `var(--color-mist)`，居中 `64px` 花盆线稿 icon（颜色 `var(--color-line)`），底部文案"轻触上传植物照片" `12px`。
- 植物信息区：紧贴 Hero 下方，padding `20px 16px 12px`。
  - 植物名：`20px/700`，颜色 `var(--color-ink)`。
  - 位置标签：`12px/400`，颜色 `var(--color-muted)`，紧跟植物名下方，间距 `4px`。
  - 状态摘要（如"3 天后浇水"）：`14px/500`，颜色 `var(--color-leaf-light)`，与植物名间距 `8px`。
- 养护任务区：
  - 区域标题 "养护任务"：`14px/600`，颜色 `var(--color-ink)`，左对齐，与信息区间距 `16px`。
  - 任务卡片：背景 `var(--color-surface)`，圆角 `12px`，padding `12px 16px`，卡片间距 `8px`，阴影 `var(--shadow-card)`。
  - 每条任务内布局：左侧完成按钮（`32px` 圆形，背景 `var(--color-gold)`，内含 checkmark `16px`），中间任务名 `14px/500` + 到期文案 `12px/400 var(--color-muted)`，右侧无操作或 kebab 菜单 icon。
  - 逾期任务：到期文案变为 `var(--color-warning)`，左侧圆形按钮边框变为 `2px solid var(--color-warning)`。
- 操作栏（编辑/添加任务）：固定于页面底部安全区上方，高度 `56px`，背景 `var(--color-surface)`，上边框 `1px solid var(--color-line)`，水平居中放置两个按钮：
  - "编辑植物"：ghost 按钮，`14px/500`，颜色 `var(--color-leaf)`。
  - "添加任务"：实心按钮，背景 `var(--color-gold)`，文字 `var(--color-ink)` `14px/600`，圆角 `12px`，高度 `40px`。

### 交互要求

- Hero 图片支持轻触全屏预览，使用系统级图片查看器交互。
- 任务完成与待办页一致：点按后卡片淡出 + 短暂 `var(--color-success)` 勾选动画。
- 下滑时 Hero 图片支持视差收缩（parallax collapse），信息区上移但操作栏始终固定。
- 无任务时显示空状态：居中文案"还没有养护任务" `14px` + "添加第一个任务"文字按钮 `var(--color-gold)` `14px/600`。

## 4.9 设置页 `/settings`

### 现状问题

- 邀请码复制失败后没有备用路径。
- 退出登录虽已加上，但还需要更明确的位置和层级。
- 家庭设置区块之间的主次关系还不够清晰。

### 调整目标

- 设置页成为"家庭协作操作页"，不是资料展示页。

### 具体调整

- 卡片顺序建议：
  1. 家庭摘要
  2. 邀请码与分享
  3. 通知权限
  4. 家庭成员
  5. 退出登录
- `复制邀请码` 按钮下方增加 `手动查看邀请码` 或失败后自动展开备用面板。
- 备用面板内提供可选中文案：
  - 邀请码本体
  - `长按可复制`
  - `你也可以直接把这串码发给家人`
- `退出登录` 放在页面底部独立区域，视觉上与普通设置项区分开。

### 视觉规格

- 页面背景：`var(--color-paper)`。
- 设置卡片：背景 `var(--color-surface)`，圆角 `16px`，padding `16px`，卡片间距 `12px`。
- 邀请码显示：使用 monospace 字体（`"SF Mono", "Fira Code", monospace`），字号 `28px/700`，颜色 `var(--color-leaf)`，letter-spacing `0.12em`，居中展示。
- 复制按钮：宽度 `100%`，高度 `44px`，背景 `var(--color-gold)`，文字 `var(--color-ink)` `14px/600`，圆角 `12px`。
- 备用面板：背景 `var(--color-mist)`，圆角 `12px`，padding `16px`，边框 `1px dashed var(--color-line)`。
- 退出登录按钮：宽度 `100%`，高度 `44px`，背景透明，边框 `1px solid var(--color-error)`，文字 `var(--color-error)` `14px/500`，圆角 `12px`。退出按钮上方留 `32px` 间距与其他设置项视觉分离。
- 家庭成员列表：每行高 `56px`，左侧 `32px` 头像圆形，中间名字 `14px/500`。

### 交互要求

- 复制成功：显示短暂正反馈（按钮变为 `var(--color-success)` + "已复制" 文案，`1.5s` 后恢复）。
- 复制失败：立即展示手动复制面板，不让用户停在失败状态。
- 退出登录：需要明确按钮文案，但不额外增加复杂确认弹窗。

## 5. 共享组件级改造建议

## 5.1 Header / Hero 模式统一

- 认证后页面取消过高的大标题 Hero。
- 标题区高度控制在“标题 + 一行说明 + 一个主按钮”的级别。
- 只有登录页和极少数空状态页面允许较强氛围背景。

## 5.2 Card 系统统一

- 卡片统一 16px 左右圆角。
- 植物卡偏照片驱动。
- 任务卡偏信息驱动。
- 设置卡偏操作驱动。

不要再让所有卡片都长得一样。

## 5.3 按钮层级

- 每屏只有一个真正的主按钮。
- 次按钮更轻，避免和主按钮同等视觉重量。
- 危险操作使用独立区域，不混在主信息块里。

## 5.4 状态反馈

- 加载：用简短中文，不用大段“正在为你准备...”。
- 错误：给下一步动作，不只提示失败。
- 空状态：明确告诉用户下一步做什么。

## 6. 实施顺序建议

建议按以下顺序落地，避免样式返工：

1. 文案统一与可访问名称同步
2. 邀请码复制修复与备用交互
3. App Shell 与底部导航的任务优先化
4. 植物列表 / 植物详情的照片驱动改版
5. 设置页 / 待办页 / onboarding 页面的整体节奏统一

> **仲裁规则**：当本文档顺序建议与 `frontend-refresh.json` 的 `priority` 字段不一致时，以 JSON 文件为准。JSON 的优先级已综合依赖关系与工程风险做过评估，本节仅提供方向性参考。
