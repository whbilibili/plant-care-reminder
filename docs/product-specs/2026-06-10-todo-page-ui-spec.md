## 0. 设计基调（与 PRD 同源）

待办页是**高频执行页**，UI 必须服务 PRD 的两个核心价值：**降低决策成本**与**消除焦虑、给正反馈**。

由此确立本页的视觉性格：**「温和的园艺工作台」**——不是冷冰冰的待办清单（to-do app 的 KPI 感），也不是花哨的营销页，而是像一张铺在窗台边的园艺记事卡：底色温润（米白 `--color-paper`）、主色沉静（深绿 `--color-leaf`）、点缀克制（暖金 `--color-gold` 只在「正反馈」时刻出现）。

三条 UI 设计原则：

1. **焦点单一**：一屏之内,「今天该干嘛」永远是视觉重心,其余信息逐级弱化。
2. **情绪由色彩承载,而非文案吼叫**：逾期用暖橙(`--color-warning`)而非警报红,完成用暖金(`--color-gold`)给奖赏感。
3. **动效服务确认,不服务炫技**：每个动效都对应一次「操作被系统接住了」的反馈,且尊重 `prefers-reduced-motion`。

## 1. 整体布局结构

待办页自上而下分四层,垂直堆叠,移动端单列：

```
┌─────────────────────────────┐
│  ① 状态化情感反馈区 (Greeting)   │  ← 随状态变化的问候卡,替代 Banner
├─────────────────────────────┤
│  ② 逾期区 (Overdue)            │  ← 主区,最高优先级
│     · 任务卡 · 任务卡 …         │
├─────────────────────────────┤
│  ③ 今天到期区 (Today)          │  ← 主区
│     · 分组标题(可一键完成)       │
│     · 任务卡 · 任务卡 …         │
├─────────────────────────────┤
│  ④ 即将到期区 (Upcoming)       │  ← 次区,视觉弱化/可折叠
│     · 灰态任务卡 …              │
└─────────────────────────────┘
        [ 固定底部导航 ]            ← Phase 1 已实现的 fixed 三 Tab
```

布局要点：

- **区段间用「分区标题 + 留白」分隔**,不用粗重分割线,保持记事卡的轻盈。
- **主区(②③)与次区(④)之间需要明显的视觉落差**：次区整体降低对比度（灰态)、缩小卡片视觉权重,让用户的眼睛自然停在主区。
- 页面顶部留白 + 反馈区,使首屏不会一上来就是密集任务,缓解「被任务追」的压迫感。

---

## 2. ① 状态化情感反馈区（设计重点）

这是替代 Banner 的核心创新区,也是「正反馈闭环」的载体。它是一张随状态变化的问候卡,而非静态图。

### 2.1 三种状态的视觉处理

| 状态 | 背景 | 主文案 | 视觉点缀 |
|------|------|--------|---------|
| **全部完成** | `--gradient-botanical`（清透绿米渐变）+ 右上角 `--gradient-accent` 光晕 | 「🌿 今天的植物都照顾好啦」 | 暖金 `--color-gold` 小装饰（叶子/对勾）、可加一株安静的插画 |
| **有今日任务** | `--color-mist`（淡雾绿）平涂 | 「今天要照顾 N 株植物」 | 无强点缀,克制 |
| **有逾期** | `--color-mist` 打底,左侧一道暖橙 `--color-warning` 细色条 | 「有 N 株植物在等你」 | **不用红色警报**,色条而非整块橙,温和提醒 |

> 设计意图：**完成态是唯一允许「华丽」的时刻**——用渐变 + 暖金 + 光晕给足奖赏感（对留存关键）；有任务态保持克制,把视觉预算让给下方任务卡。

### 2.2 反馈区交互

- 反馈区**不可点击**(纯信息),避免与下方任务卡的可操作性混淆。
- 文案中的数字(N 株)实时随完成/推迟更新,数字变化时可加一个轻微的数字滚动/淡入(可选,服务于「我刚做的事被记下了」)。

## 3. ②③ 主区任务卡设计

延续现有 `DueTaskCard` 结构（缩略图 + 名称/类型/到期 + 完成按钮），在其上做三处 UI 增强：

### 3.1 任务类型色条 / 图标（降低决策成本）

卡片左侧引入**任务类型识别**,让用户扫一眼知道「要干嘛」,而非读文字：

| 任务类型 | 颜色方向 | 图标 |
|---------|---------|------|
| 浇水 watering | 水蓝（冷色，区别于主绿） | 💧 |
| 施肥 fertilizing | 暖棕/土色 | 🌱 |
| 喷雾 misting | 青绿 | 💨 |
| 换盆 repotting | 陶土橙 | 🪴 |
| 修剪 pruning | 叶绿 | ✂️ |
| 自定义 custom | 中性灰 | 🏷️ |

> 实现建议：在卡片缩略图与内容之间,或缩略图角标,放一个 4px 宽的类型色条 / 小圆点图标。色值后续在 token 里补充 `--color-task-*` 系列,保持「禁止硬编码」约定。

### 3.2 逾期视觉（沿用并强化）

- 现有逾期 pill（暖橙 `rgba(221,107,32,0.08)` 底 + `--color-warning` 字）方向正确,**保留**。
- 按 PRD §1.2,逾期天数越多越靠前；文案体现「逾期 N 天」,色彩仍走暖橙不升级为红,贯彻「不施压」。

### 3.3 完成按钮与微交互

- 完成按钮沿用 `appearance="circle"` 圆形勾选。
- **完成微交互（高频留存隐藏武器）**：打勾瞬间 → 卡片内一个水滴/叶子小动画 + 卡片以 300ms 淡出移出主区（现有 `completedCardStyle` 已做淡出,在此基础上加「奖赏动效」）。
- 移动端配合 **iOS Haptic 轻震**,把「打勾」变成小确幸。
- 所有动效遵守 `prefers-reduced-motion`：减弱时仅保留即时状态切换,去除动画。

## 4. ③ 分区级一键完成（对应 PRD §9.3）

- 在「今天到期」区,若同类型任务 ≥2 个,在**类型聚合的分组标题栏右侧**放一个「✅ 全部完成」轻按钮（文字按钮 / 弱描边,不抢主完成按钮的视觉）。
- 点击后该组卡片**批量淡出**,并触发统一的撤销浮条（见 §6）。
- 视觉上「全部完成」按钮用 `--color-leaf-light` 描边/文字,低于单卡完成按钮的视觉权重,避免误触主导。

---

## 5. ④ 即将到期区（弱化处理）

- 整区**降低视觉权重**：卡片背景仍是 surface,但文字色统一降到 `--color-muted`,缩略图可降饱和。
- 已完成但下次落在 1–3 天的任务（PRD §1.3 方案 A）在此区以**灰态**显示,带「✅ 已完成 · 下次 X 月 X 日」标注,文字走 `--color-muted`,左侧可加一个淡绿对勾,传达「这个不用管了,心里有数即可」。
- 该区**支持折叠**（默认展开,标题可点收起）,给重度用户清爽视图的选项。

---

## 6. 推迟与撤销的交互视觉

### 6.1 推迟入口（对应 PRD §8）

- 任务卡上提供「推迟」次级入口（如卡片右滑露出,或卡片内一个轻量「推迟 1 天」文字按钮）,视觉权重低于「完成」。
- 点击推迟时弹出**轻确认**(底部 sheet 或行内提示),展示 PRD §8.2 的告知文案：「该任务已逾期 N 天,推迟后将从今天起顺延,下次提醒 X 月 X 日」。文案区用 `--color-muted`,语气温和。
- 连续推迟达阈值（§8.4）时,卡片上出现一条温和建议条(淡雾绿底 `--color-mist`),文案「这株植物最近总被推迟,要不要调整养护周期?」,提供跳转养护设置的链接。

### 6.2 撤销浮条（对应 PRD §9.1）

- 完成/批量完成后,屏幕底部(导航栏上方)浮出一条 3–5 秒的撤销条。
- 视觉：深绿 `--color-leaf` 底 + 米白文字 + 「撤销」文字按钮（暖金 `--color-gold` 强调）,圆角 `--radius-pill` 或 `--radius-sheet`。
- 文案：单个「已完成,可撤销」/ 批量「已完成 5 个浇水任务」。
- 浮条用 `--shadow-sheet` 投影,从底部滑入,超时滑出。

---

## 7. 空状态（对应 PRD §4.4）

- 无任务时不白屏,复用现有 `EmptyState` 组件,但区分两种语境：
  - **新家庭空间(还没植物)**：温暖插画 + 引导「去添加第一株植物」（onboarding,决定留存）。
  - **今天无任务(有植物但都做完了)**：复用 §2.1「全部完成」反馈区 + 「今天无需养护,去看看你的植物」。
- 插画风格走 botanical 线条手绘感,配色 `--color-mist` / `--color-leaf-light`,不喧宾夺主。

## 8. 配色与字体应用总表

| 用途 | Token | 说明 |
|------|-------|------|
| 页面底色 | `--color-paper` | 温润米白,记事卡基底 |
| 卡片表面 | `--color-surface` | 纯白,任务卡 |
| 弱化区底 | `--color-mist` | 雾绿,反馈区/弱区 |
| 主文字/标题 | `--color-ink` / `--color-leaf` | 深绿系 |
| 次要文字 | `--color-muted` | 即将到期区、辅助说明 |
| 逾期提示 | `--color-warning` | 暖橙,不升级为红 |
| 正反馈/奖赏 | `--color-gold` | 仅在完成态、撤销按钮出现 |
| 完成态成功 | `--color-success` | 灰态对勾 |
| 标题字体 | `--font-heading`（Lora） | 反馈区主文案可用,增加温度 |
| 正文字体 | `--font-body`（Raleway） | 任务卡、说明文字 |

> 强约束（沿用 tokens.css）：**所有色值、间距、圆角、阴影一律引用 token 变量,禁止硬编码 #hex**。新增的任务类型色（§3.1）需先在 tokens.css 补 `--color-task-*` 再使用。

## 9. 动效原则

| 时刻 | 动效 | 时长/曲线 | 目的 |
|------|------|----------|------|
| 完成单任务 | 水滴/叶子微动 + 卡片淡出移出 | ~300ms ease-out | 奖赏 + 确认移出 |
| 批量完成 | 多卡依次淡出（轻 stagger） | 各 ~200ms,错峰 | 「一次清掉一类」的爽感 |
| 撤销浮条 | 底部滑入 / 超时滑出 | ~250ms | 提示可补救 |
| 反馈区数字变化 | 数字淡入/滚动 | ~200ms | 「我做的被记下了」 |
| 全部完成态 | 反馈区光晕轻微呼吸（可选） | 缓慢循环 | 满足感 |

- 全部动效在 `prefers-reduced-motion: reduce` 下退化为即时状态切换（tokens.css 已有全局兜底）。
- 优先 CSS 动画；复杂编排（stagger）可按需引入,但保持轻量。
