# 植物养护提醒应用 MVP 需求文档

**版本**：v0.1  
**日期**：2026-06-05  
**定位**：家庭内部使用的植物养护提醒工具  
**推荐技术方向**：`PWA + Convex`

---

## 1. 产品目标

为家庭成员提供一个共享的植物养护提醒工具，帮助大家管理家里的花和绿植，避免忘记浇水、施肥等日常养护事项。

MVP 只解决 3 个核心问题：

1. 知道家里有哪些植物，以及每株植物的基本资料。
2. 知道每株植物下一次什么时候该浇水或施肥。
3. 任意家庭成员完成一次养护后，系统自动计算下一次提醒时间，并同步给全家。

---

## 2. 目标用户

- 家庭内部成员
- 主要使用设备为 `iPhone`
- 使用人数较少，通常为 2 到 6 人
- 对植物养护有日常需求，但不希望靠手工记忆或纸质记录

---

## 3. MVP 范围

### 3.1 In Scope

MVP 必须包含以下能力：

- 家庭成员独立账号登录
- 创建或加入同一个家庭空间
- 植物资料管理
- 植物图片上传
- 为每株植物配置一个或多个养护任务
- 养护任务支持按“间隔天数”提醒
- 到期提醒
- 点击“完成”后自动顺延到下一次提醒时间
- 家庭成员之间实时同步最新状态

### 3.2 Out of Scope

MVP 明确不做以下内容：

- AI 识别植物
- 病虫害诊断
- 根据天气自动调整养护周期
- 复杂重复规则（如每月第一个周三）
- 离线优先
- 原生 iOS App
- 电商购肥/购土能力
- 植物社区、分享、公开内容

---

## 4. 核心使用场景

### 场景 1：录入植物

用户首次使用时，可以添加家里的植物，并填写：

- 名称
- 图片
- 描述
- 备注
- 摆放位置（可选，如客厅、阳台）

### 场景 2：设置养护规则

用户可以为某株植物添加养护任务，例如：

- 浇水：每 5 天一次
- 施肥：每 20 天一次
- 喷雾：每 3 天一次

每个任务至少包含：

- 任务类型
- 间隔天数
- 上次完成时间
- 下次提醒时间

### 场景 3：收到提醒并完成任务

当任务到期时，系统向家庭成员发出提醒。用户打开应用后，可以看到：

- 哪株植物需要处理
- 需要做什么任务
- 是否已经逾期

用户点击“完成”后，系统自动执行：

- 将本次时间记录为最新完成时间
- 按任务间隔天数重算下一次提醒时间
- 将新的状态同步给全家

### 场景 4：家庭共享协作

多个家庭成员加入同一个家庭空间后：

- 看到同一批植物
- 看到同一批养护任务
- 任意一人完成任务后，其他人看到的状态立即更新

## 5. MVP 功能需求

## 5.1 账号与家庭空间

系统应支持：

- 用户注册与登录
- 用户创建家庭空间
- 用户通过邀请方式加入已有家庭空间
- 一个用户在 MVP 内只属于一个家庭空间

MVP 验收标准：

- 新用户可以注册并进入应用
- 家庭管理员可以创建家庭空间
- 家庭成员可以加入同一家庭空间
- 同一家庭中的成员能看到一致的数据

## 5.2 植物管理

每株植物至少包含以下字段：

- 名称
- 图片
- 描述
- 备注
- 摆放位置（可选）
- 创建时间
- 创建人

系统应支持：

- 新增植物
- 编辑植物信息
- 查看植物详情
- 删除或归档植物

MVP 验收标准：

- 用户可以添加一株带图片的植物
- 用户可以在列表页看到植物名称和封面图
- 用户可以进入详情页查看完整资料

## 5.3 养护任务管理

每株植物可以配置多个养护任务。

任务类型包括：

- 浇水
- 施肥
- 喷雾
- 换土
- 修剪
- 自定义任务

每个任务至少包含：

- 任务类型
- 间隔天数
- 是否启用
- 上次完成时间
- 下次提醒时间

系统应支持：

- 添加任务
- 编辑任务规则
- 启用/停用任务
- 删除任务

MVP 验收标准：

- 用户可以为同一株植物添加多个任务
- 不同任务可以有不同间隔天数
- 保存后系统能正确展示下次提醒时间

## 5.4 到期提醒

系统应在任务到期时提醒家庭成员。

MVP 提醒策略：

- 首选 `PWA Web Push`
- 用户需要将网页添加到 iPhone 主屏幕，并授予通知权限

提醒内容至少包含：

- 植物名称
- 任务类型
- 到期状态

MVP 验收标准：

- 到期任务会出现在应用内待办列表
- 已开启通知的用户能收到手机提醒
- 点击提醒后可进入对应植物或任务详情

## 5.5 完成任务与自动顺延

用户点击任务“完成”后，系统自动：

- 记录本次完成时间
- 更新上次完成时间
- 按间隔天数重算下次提醒时间

计算规则：

- `下次提醒时间 = 本次完成时间 + 间隔天数`

MVP 验收标准：

- 点击完成后页面立即更新
- 家庭其他成员端同步看到最新状态
- 下次提醒时间计算正确

## 5.6 待办视图

系统应提供一个集中待办视图，用于展示：

- 今日到期任务
- 已逾期任务
- 即将到期任务（可选，若实现成本低）

MVP 验收标准：

- 用户打开应用后，能快速看到需要处理的植物任务
- 待办列表按紧急程度排序，逾期优先

## 6. 非功能性需求

### 6.1 设备与平台

- 首要目标平台为 `iPhone`
- 页面需适配手机竖屏
- 家庭成员无需安装 App Store 原生应用即可使用

### 6.2 同步体验

- 任意成员完成任务后，其他成员应在短时间内看到同步结果
- 不允许出现“同一个任务一边已完成、一边仍显示待处理”长期不一致的状态

### 6.3 易用性

- 常用操作应尽量控制在 3 步内完成
- 用户从提醒进入应用后，应能在短路径内完成任务打勾

### 6.4 成本

- 优先采用免费或低成本可用方案
- 允许数据先存放于云端，只要部署和维护成本低

---

## 7. 推荐 MVP 信息结构

MVP 建议包含以下页面：

1. 登录 / 注册页
2. 创建 / 加入家庭页
3. 植物列表页
4. 植物详情页
5. 新增 / 编辑植物页
6. 新增 / 编辑任务页
7. 待办页
8. 家庭设置页

---

## 8. 数据对象草案

MVP 至少需要以下核心对象：

- `User`
- `Family`
- `FamilyMember`
- `Plant`
- `PlantTask`
- `TaskCompletionLog`
- `PushSubscription`

对象关系：

- 一个 `Family` 有多个 `FamilyMember`
- 一个 `Family` 有多株 `Plant`
- 一株 `Plant` 有多个 `PlantTask`
- 一个 `PlantTask` 有多条 `TaskCompletionLog`

---

## 9. MVP 成功标准

满足以下条件，可认为 MVP 达到可试用状态：

1. 家庭中至少 2 个成员可以登录并共享同一批植物数据。
2. 用户可以添加植物图片、名称、描述和备注。
3. 用户可以为植物配置浇水或施肥间隔。
4. 到期任务能在应用内正确显示。
5. 用户完成任务后，系统能自动计算下一次提醒时间。
6. 完成结果能同步给其他家庭成员。
7. 至少一位已授权通知的 iPhone 用户可以收到提醒。

---

## 10. 当前推荐技术方向

MVP 推荐采用：

- 前端：`React PWA`
- 后端与数据库：`Convex`
- 图片存储：`Convex File Storage`
- 认证：`Convex Auth` 或兼容的外部认证方案
- 部署形态：前端静态部署 + Convex Cloud

推荐理由：

- 适合家庭共享和实时同步
- 支持文件存储和定时任务
- 开发成本低于原生 App
- 如果后续 iPhone 推送体验不够理想，可逐步升级为 `Expo / React Native + Convex`

---

## 11. 后续版本候选

以下内容建议在 MVP 验证通过后再考虑：

- 跳过一次 / 延后几天
- 养护历史时间线
- 植物分组与房间筛选
- 更丰富的提醒规则
- 多图上传
- 施肥配方与用量备注
- 原生 App 化
