## 0. 设计基调

设置页是**低频辅助页**，UI 必须服务 PRD 的三个核心价值：**协作入口**、**自我掌控**、**可信兜底**。

由此确立本页的视觉性格：**「家庭事务台」**——不是花哨的展示页，也不是冷冰冰的后台管理表单，而是一张温润、分区明确的事务台：用「分组小标题 + 卡片」建立秩序感，用图标 chip 让每个区块一眼可辨，用一张品牌绿头图卡承载"家庭"这个情感主体，把破坏性操作收进统一的确认 sheet 里，安全且克制。

三条 UI 设计原则：

1. **分区清晰，秩序优先**：用「个人 / 家庭 / 通知与应用」三组小标题切分，用户找东西沿组定位，不在一堆白卡里大海捞针。
2. **一个情感锚点，其余安静**：仅家庭摘要卡用品牌绿做视觉锚点，其余卡片均为安静白卡，避免页面"满屏都是重点"。
3. **危险操作显性但收敛**：移除、重置、退出都可见、不埋深，但点击后统一进二次确认 sheet，杜绝误触。

## 1. 整体布局结构

设置页自上而下：页面标题 + 三个语义分组，移动端单列垂直堆叠：

```
┌─────────────────────────────────────┐
│  设置                                │  ← ① 页面标题 h1（全页唯一）
├─────────────────────────────────────┤
│  个人                                │  ← ② 分组小标题
│   ┌───────────────────────────────┐ │
│   │ 👤  我的称呼          小柯  ›   │ │  ← 设置项（可点编辑）
│   ├───────────────────────────────┤ │
│   │ ✉️  我的账号   ke@example.com  │ │  ← 设置项（只读）
│   └───────────────────────────────┘ │
│   ┌───────────────────────────────┐ │
│   │           退出登录             │ │  ← 危险操作（描边按钮）
│   └───────────────────────────────┘ │
├─────────────────────────────────────┤
│  家庭                                │  ← ② 分组小标题
│   ┌───────────────────────────────┐ │
│   │ 🏠  家庭头图卡（品牌绿渐变）     │ │  ← ③ 家庭摘要（情感锚点）
│   └───────────────────────────────┘ │
│   ┌───────────────────────────────┐ │
│   │ 🔑  邀请码卡（含复制 / 重置）    │ │  ← ④ 邀请码卡
│   └───────────────────────────────┘ │
│   ┌───────────────────────────────┐ │
│   │ 👥  成员（N）含头像 / 移除      │ │  ← ⑤ 成员卡
│   └───────────────────────────────┘ │
├─────────────────────────────────────┤
│  通知与应用                          │  ← ② 分组小标题
│   ┌───────────────────────────────┐ │
│   │ 🔔  设备通知（沿用现有卡）       │ │  ← ⑥ 通知卡
│   │      ▸ 收不到提醒？（排障）      │ │  ← 排障折叠
│   └───────────────────────────────┘ │
│   ┌───────────────────────────────┐ │
│   │ ℹ️  关于 · 版本 v0.2.0          │ │  ← ⑦ 关于卡
│   └───────────────────────────────┘ │
└─────────────────────────────────────┘
          [ 固定底部导航 ]                ← 三 Tab：待办 / 植物 / 设置
```

布局要点：

- 页面顶部 **页面标题「设置」(h1)** 与待办/植物页标题层级一致，全页仅此一个 h1。
- **分组之间用 `--space-lg`（24px）间距**分隔；组内卡片之间用 `--space-md`（16px）。
- **分组小标题与该组首个卡片之间用 `--space-sm`（8px）**，让小标题"贴住"它统辖的内容。
- 整页外层沿用现有 `pageStyle`：`display: grid; gap: --space-md`，本次在其上增加分组容器。
- 页面底部预留 `80px` 安全区（底部导航 + 安全距离）。

## 2. ① 页面标题与 ② 分组小标题

### 2.1 页面标题

| 元素 | 规格 |
|------|------|
| 文字 | 「设置」 |
| 字体 | `--font-heading`（Lora），24px，font-weight 700 |
| 颜色 | `--color-ink` |
| 行高 | 1.2 |
| 下间距 | `--space-md`（16px） |

### 2.2 分组小标题

分组小标题是本次重组的关键视觉元素，必须**克制**——它是路标，不是大标题。

| 元素 | 规格 |
|------|------|
| 文字 | 「个人」/「家庭」/「通知与应用」 |
| 字体 | `--font-body`（Raleway），13px，font-weight 700 |
| 颜色 | `--color-leaf-light` |
| 字距 | `letter-spacing: 0.06em` |
| 大小写 | 中文无大小写，**不**做 uppercase 处理 |
| 左内边距 | 与卡片左缘对齐或缩进 4px |

设计意图：分组小标题用 `--color-leaf-light` 而非 `--color-ink`，刻意比卡片标题更轻，避免与卡内 h2 抢层级；它的作用是"轻轻提示这里是哪一组"。

## 3. 卡片与设置项通用规格

设置页有两种容器：**卡片（Card）** 和 **设置项行（Setting Row）**。

### 3.1 卡片容器（沿用现有 `cardStyle`）

| 属性 | 值 |
|------|-----|
| 背景 | `--color-surface` |
| 圆角 | `--radius-card`（16px） |
| 边框 | 1px solid `--color-line` |
| 阴影 | `--shadow-card` |
| 内边距 | `--space-md`（16px） |
| 内部布局 | `display: grid; gap: --space-sm` |

### 3.2 卡片标题区（图标 chip + eyebrow + h2）

本次新增**图标 chip**，替代现有纯文字 eyebrow，让每个区块一眼可辨：

```
┌──────┐
│  🏠  │  家庭          ← chip + eyebrow 同行
└──────┘  我们的小花园    ← h2 标题
```

| 元素 | 规格 |
|------|------|
| 图标 chip 尺寸 | 32×32 |
| chip 圆角 | `--radius-button`（12px） |
| chip 背景 | `--color-mist`（普通卡）；家庭头图卡见 §4 |
| chip 图标 | emoji 或 SVG，18px，居中 |
| eyebrow 文字 | `--font-body`，12px，font-weight 700，`--color-leaf-light`，字距 0.08em |
| h2 标题 | `--font-heading`，16px，font-weight 600，`--color-ink`，行高 1.25 |
| chip 与文字间距 | `--space-sm`（8px） |

> 图标映射：个人项无独立卡（用行内小图标）；家庭 🏠、邀请码 🔑、成员 👥、通知 🔔、关于 ℹ️。

### 3.3 设置项行（Setting Row）

用于「个人」组的"我的称呼""我的账号"，是横向的标签—值结构：

```
┌────────────────────────────────────┐
│ 👤  我的称呼              小柯   ›   │
└────────────────────────────────────┘
```

| 属性 | 值 |
|------|-----|
| 布局 | `display: flex; align-items: center` |
| 最小高度 | 52px |
| 内边距 | 12px 16px |
| 行内图标 | 20px，`--color-leaf-light`，右间距 12px |
| 标签文字 | `--font-body`，15px，font-weight 500，`--color-ink` |
| 值文字 | `--font-body`，15px，`--color-muted`，`margin-left: auto` |
| 可点击项尾部 | `›` 箭头（chevron），`--color-line`，14px |
| 只读项 | 无箭头，值不可点击 |
| 多项分隔 | 项之间 1px `--color-line` 内分割线（同一卡内） |

## 4. ③ 家庭头图卡设计（情感锚点）

家庭摘要从普通白卡升级为**全页唯一的品牌绿视觉锚点**，承载"家庭"这个情感主体。

### 4.1 结构

```
┌─────────────────────────────────────┐
│ ┌──────┐                      🌿     │
│ │  🏠  │  家庭                       │  ← chip（白底）+ eyebrow（浅色）
│ └──────┘                            │
│                                     │
│  我们的小花园                        │  ← 家庭名（白色大字）
│  3 位家人 · 一起照顾 8 盆植物         │  ← 摘要副文案
└─────────────────────────────────────┘
```

### 4.2 视觉规格

| 属性 | 值 |
|------|-----|
| 背景 | `--color-leaf` 实底 或 `linear-gradient(135deg, var(--color-leaf), var(--color-leaf-light))` |
| 圆角 | `--radius-card`（16px） |
| 边框 | 无 |
| 阴影 | `--shadow-card` |
| 内边距 | `--space-md`（16px） |
| 装饰 | 右上角一枚半透明 🌿（48px，opacity 0.18，溢出裁切），作为植物气质点缀 |
| chip 背景 | 白色半透明 `rgba(255,255,255,0.18)` |
| chip 图标 | 🏠，白色 |
| eyebrow | 12px，700，`rgba(255,255,255,0.75)`，字距 0.08em |
| 家庭名 | `--font-heading`，22px，700，`--color-paper`（近白），行高 1.2 |
| 副文案 | `--font-body`，13px，`rgba(255,255,255,0.85)`，行高 1.5 |

### 4.3 文案规则

| 场景 | 副文案 |
|------|--------|
| 仅自己 1 人 | 「还没有家人加入，把邀请码发出去吧」 |
| 2 人及以上 | 「N 位家人 · 一起照顾这片植物」（株数可选拼接） |

设计意图：用一张深绿渐变卡把页面视觉重心从"白卡海"里拎出来，给低频的设置页一个温度入口。这是全页**唯一**的强色块（与植物列表页"逾期色条是唯一彩色信号"同理）。

## 5. ④ 邀请码卡设计

承接现有邀请码卡，保留复制 fallback，新增"重置"入口（仅管理员可见）。

### 5.1 结构

```
┌─────────────────────────────────────┐
│ ┌──────┐                            │
│ │  🔑  │  邀请码                     │
│ └──────┘  把它发给家人，一起照顾植物   │
│                                     │
│   ┌─────────────────────────────┐   │
│   │  A B 3 K 9 P     [ 📋 复制 ] │   │  ← 邀请码高亮容器
│   └─────────────────────────────┘   │
│                                     │
│   重置邀请码                          │  ← 管理员可见，文字链（次级）
└─────────────────────────────────────┘
```

### 5.2 邀请码高亮容器

| 属性 | 值 |
|------|-----|
| 背景 | `--color-mist` |
| 边框 | 1px dashed `--color-line` |
| 圆角 | `--radius-input`（12px） |
| 内边距 | 12px 16px |
| 布局 | `display: flex; align-items: center; justify-content: space-between` |
| 邀请码文字 | `--font-mono`，24px，700，`--color-leaf`，字距 0.12em，`user-select: all` |
| 复制按钮 | 文字 + 📋 图标，`--color-leaf`，14px 600；复制成功切换为「已复制」+ `--color-success` |

设计意图：把邀请码放进虚线描边容器，强化它"这是要分享出去的东西"的语义；复制按钮内嵌在容器内，紧贴邀请码，符合"看到码就想复制"的动线。

### 5.3 重置邀请码入口

| 属性 | 值 |
|------|-----|
| 可见性 | 仅 `role === "admin"` 渲染；普通成员不渲染（不置灰） |
| 形态 | 文字链（次级），`--font-body`，13px，font-weight 500，`--color-muted` |
| 位置 | 邀请码容器下方，左对齐 |
| 点击 | 触发"重置邀请码"确认 sheet（§8） |

### 5.4 复制失败 fallback（不回归）

沿用现有规范：复制失败时自动展开 `--color-mist` 底、1px dashed 边的备用区，居中展示可选中邀请码 + 「长按可复制」+ 「也可以直接把这串码发给家人」。

## 6. ⑤ 成员卡设计

承接现有 `MembersList`，新增**头像占位**与**管理员移除入口**，并消除重复命名。

### 6.1 卡片标题（消除重复命名）

- eyebrow：「家庭」
- h2：「成员（N）」（N 为成员数）
- 不再出现 eyebrow 与标题同写"家庭成员"。

### 6.2 成员行结构

```
┌─────────────────────────────────────┐
│ ⓚ  小柯  [我]                管理员   │  ← 头像 + 名字 + 自己标 + 角色徽章
├─────────────────────────────────────┤
│ ⓜ  妈妈                  成员  [移除] │  ← 管理员视角才有「移除」
└─────────────────────────────────────┘
```

| 元素 | 规格 |
|------|------|
| 行布局 | `display: flex; align-items: center; gap: 12px` |
| 行内边距 | 12px |
| 行间分隔 | 1px `--color-line` 内分割线（同卡内） |

### 6.3 头像占位

设计规范早已要求"成员项展示头像占位"，本次落地：

| 属性 | 值 |
|------|-----|
| 尺寸 | 36×36，圆形（`--radius-pill`） |
| 内容 | 显示名首字（中文取首字，英文取首字母大写） |
| 字体 | `--font-body`，15px，700，`--color-surface`（白字） |
| 底色 | 按名字 hash 取一组**柔和品牌色**轮换（见下） |
| 兜底 | 无名字时显示 👤，底色 `--color-muted` |

头像底色取色池（均为低饱和、与 botanical 体系协调）：`--color-leaf`、`--color-leaf-light`、`--color-task-misting`、`--color-task-pruning`、`--color-task-fertilizing`。用 `displayName` 的字符和取模选色，保证同一人颜色稳定。

### 6.4 名字与角色区

| 元素 | 规格 |
|------|------|
| 名字 | `--font-body`，15px，font-weight 600，`--color-ink`，单行省略 |
| 邮箱副行（名字≠邮箱时） | `--font-body`，12px，`--color-muted` |
| 「我」标 | pill，`--radius-pill`，`--color-mist` 底 + `--color-leaf` 字，11px 700 |
| 管理员徽章 | pill，**`--color-leaf` 实底 + `--color-paper` 字**（实心强调） |
| 成员徽章 | pill，`--color-mist` 底 + `--color-leaf-light` 字，1px `--color-line` 描边（弱化） |

设计意图：管理员实心、成员描边，用填充差异表达权限层级，比文字更快可辨。

### 6.5 移除入口

| 属性 | 值 |
|------|-----|
| 可见性 | 仅当前用户为管理员、且目标行非自己、非创建者时渲染 |
| 形态 | 文字链「移除」，`--font-body`，13px，font-weight 500，`--color-error` |
| 位置 | 行尾，`margin-left: auto`（与角色徽章并列时居右） |
| 热区 | 最小 44×44 |
| 点击 | 触发"移除成员"确认 sheet（§8），需 `stopPropagation` |

## 8. 确认 Sheet（统一规范）

退出登录 / 移除成员 / 重置邀请码三者共用**底部确认 sheet**，复用现有 `PostponeConfirmSheet` 同款动效（`sheet-overlay-in/out` + `sheet-rise-in/sink-out`）。

### 8.1 Sheet 结构

```
        ─────────                       ← 顶部抓手（grabber）
   重置邀请码？                          ← 标题
   旧邀请码将立即失效，已发出去的旧码       ← 说明（点名后果）
   将无法再加入家庭。

   ┌─────────────────────────────────┐
   │           重置邀请码             │  ← 主操作按钮
   └─────────────────────────────────┘
   ┌─────────────────────────────────┐
   │             取消                │  ← 取消按钮
   └─────────────────────────────────┘
```

### 8.2 Sheet 容器规格

| 属性 | 值 |
|------|-----|
| 遮罩 | `--color-overlay-scrim`，`sheet-overlay-in` 进 / `sheet-overlay-out` 出 |
| 容器背景 | `--color-surface` |
| 圆角 | 顶部 `--radius-sheet`（20px），底部贴边 |
| 阴影 | `--shadow-sheet` |
| 入场 | `sheet-rise-in`（带回弹），出场 `sheet-sink-out` |
| 抓手 | 36×4，`--color-line`，圆角 pill，居中 |
| 内边距 | `--space-md`（16px）+ 底部安全区 |
| 标题 | `--font-heading`，18px，600，`--color-ink` |
| 说明 | `--font-body`，14px，`--color-muted`，行高 1.6 |
| 关闭 | 点遮罩、取消按钮、Esc 均可关闭 |

### 8.3 三种确认的按钮配置

| 操作 | 主按钮文案 | 主按钮样式 | 取消 |
|------|-----------|-----------|------|
| 退出登录 | 「退出登录」 | 描边态：透明底 + 1px `--color-error` 边 + `--color-error` 字 | 「取消」（ghost） |
| 移除成员 | 「移除 XXX」 | 实底危险态：`--color-error` 底 + `--color-surface` 字 | 「取消」（ghost） |
| 重置邀请码 | 「重置邀请码」 | 实底强调：`--color-leaf` 底 + `--color-paper` 字 | 「取消」（ghost） |

设计意图：移除成员是不可逆的破坏性操作，用 `--color-error` 实底最强警示；退出登录可恢复（重新登录即可），用描边态适度警示；重置邀请码非破坏数据，用品牌绿强调态而非红色。三者共用同一 sheet 骨架，差异仅在按钮配色与文案。

## 11. 响应式

设置页为单列布局，响应式策略简单：

- **移动端（< 480px，主目标）**：单列垂直堆叠，卡片宽度撑满，左右页面边距沿用现有 `pageStyle` padding。
- **中等屏（480–768px）**：内容区最大宽度 `560px` 居中，避免设置项行过长、值与标签视觉割裂。
- **桌面（> 768px）**：维持单列居中 `560px`，不做多列——设置页信息量小，多列反而降低可读性。
- 家庭头图卡、邀请码容器、成员行均使用 `flex` + `min-width: 0` + 文字省略，保证窄屏不溢出。
- 确认 sheet 在任何宽度下都贴底全宽（最大 `560px` 居中）。

## 12. 无障碍

- **页面标题** 使用语义 `<h1>`，分组小标题用 `<h2>` 或带 `role` 的容器，卡内标题用 `<h3>`，保证标题层级连续。
- **设置项可点行** 用 `<button>` 或带 `role="button"` + `tabindex=0`，可键盘聚焦、Enter 触发。
- **移除 / 重置 / 退出** 等危险操作，确认 sheet 打开时焦点移入 sheet，关闭后焦点归还触发元素（focus trap）。
- **排障折叠** 触发器带 `aria-expanded` + `aria-controls`，展开区带对应 `id`。
- **头像占位** 为装饰性，`aria-hidden="true"`，成员名字由文本独立承载，不依赖头像传达信息。
- **角色徽章 / 状态** 不单靠颜色区分：管理员/成员有文字、移除有「移除」文字，色觉障碍用户可读。
- **触控热区** 所有可点击元素 ≥ 44×44；移除文字链等小目标用透明 padding 扩展热区。
- **对比度**：家庭头图卡上的白字与深绿底对比 ≥ 4.5:1；`--color-muted` 文字仅用于 13px 以上副文案，正文关键信息用 `--color-ink`。
- **动效降级**：确认 sheet、排障展开、chevron 旋转均遵循 `prefers-reduced-motion`，开启后改为即时切换（沿用 tokens.css 现有降级规则）。

## 14. 验收标准（UI 维度）

1. 页面仅有一个 `<h1>`「设置」，三个分组小标题层级、配色符合 §2.2。
2. 个人 / 家庭 / 通知与应用 三组顺序、间距（组间 24px、组内 16px、标题贴 8px）符合 §1。
3. 家庭头图卡为全页唯一强色块，渐变 / 装饰叶 / 白字对比度符合 §4。
4. 邀请码用 Mono 24px、虚线容器、内嵌复制，复制成功反馈为「已复制」+ 成功色；失败 fallback 不回归。
5. 重置邀请码文字链仅管理员可见；普通成员不渲染（非置灰）。
6. 成员行有头像占位（首字 + 稳定取色），管理员/成员徽章用实底/描边区分，「我」标存在。
7. 移除文字链仅管理员、非自己、非创建者三条件同时满足时出现，热区 ≥ 44。
8. 退出 / 移除 / 重置三种确认共用 sheet 骨架，动效复用 `sheet-rise-in/overlay-in`，按钮配色符合 §8.3。
9. 排障折叠默认收起，`aria-expanded` 正确，展开动效遵循 reduced-motion。
10. 全部色值 / 间距 / 圆角引用 token，无新增硬编码；reduced-motion 下动效降级正常。
11. 移动端窄屏（320px）下邀请码、成员行、头图卡均不溢出、文字正确省略。
