# 季节性养护建议

**版本**：v1.0
**日期**：2026-06-23
**关联文档**：承接 [迭代储备大纲](./2026-06-17-iteration-backlog-outline.md)（P2-2）、[植物知识卡片](./2026-06-23-plant-knowledge-card-spec.md)（品种数据联动）、[养护任务灵活性](./2026-06-22-care-task-flexibility-spec.md)（季节性排期模式联动）
**文档定位**：根据当前季节在关键页面展示养护提示，帮助用户感知季节变化对植物养护的影响，并引导用户适时调整养护频率。轻量实现，以前端展示层为主。

---

## 0. 本次迭代要解决的核心问题

植物的养护需求随季节显著变化——夏天蒸发快需要多浇水，冬天休眠期需要少浇水少施肥。但 MVP 没有任何季节意识，用户设置的固定间隔全年不变。这导致两个体验缺口：

- **"入冬了还按夏天的频率浇水，结果盆土一直湿着，根都烂了"** —— 缺乏季节提醒：用户忘记在换季时调整养护频率，导致过度浇水或施肥。
- **"每次换季都要一盆一盆去改间隔天数，太麻烦了"** —— 缺乏批量调整引导：即使用户意识到需要换季调整，操作路径也很长（逐个编辑每盆植物的每个任务）。

本次通过在关键页面展示**季节性养护提示**，让用户在合适的时机收到"该调整了"的信号，并提供便捷的调整入口。

---

## 1. 产品目标

在待办页和植物详情页，根据当前季节和季节转换时机，展示针对性的养护提示。对于已关联品种的植物，提供品种级别的季节性建议；对于未关联品种的植物，提供通用的季节性养护常识。同时在换季时段主动提醒用户检查和调整养护频率。

---

## 2. 用户场景

### 场景 1：夏季高温提醒

7 月盛夏，用户打开待办页，顶部出现一条季节提示横幅："夏季高温，植物蒸发加快，建议适当增加浇水频率。"用户意识到自己的绿萝还是按春天的 5 天一次在浇，于是去编辑改为 3 天一次。

### 场景 2：入冬换季提醒

11 月初（秋冬交替），用户打开待办页，看到换季提示："即将进入冬季，大多数植物进入休眠期，建议减少浇水和施肥频率。"提示下方有"查看调整建议"按钮，点击后展示当前家庭中各植物的建议调整方案。

### 场景 3：品种级别的季节建议

用户查看琴叶榕的详情页，知识卡片的"当季要点"区域显示："秋冬养护 · 琴叶榕在冬季需减少浇水至每 10~14 天一次，停止施肥，避免冷风直吹。"这是基于品种库中琴叶榕的 `seasonalNotes.autumnWinter` 字段展示的。

### 场景 4：已使用季节性排期的用户

用户的发财树已经配置了季节性排期模式（P1-2），系统自动按季节切换间隔。此时详情页的季节提示标注"已启用季节性排期，系统将自动调整"，不再建议手动修改。

## 3. 功能设计

### 3.1 季节定义

与养护任务灵活性（P1-2）保持一致的季节划分：

| 季节 | 月份范围 | 养护特征 |
|------|---------|---------|
| 春夏（生长季） | 3月~8月 | 生长旺盛，需水量大，可施肥 |
| 秋冬（休眠季） | 9月~2月 | 生长放缓或休眠，需水量减少，减少或停止施肥 |

**换季时段**定义：

- 春夏→秋冬过渡：9月1日~9月15日。
- 秋冬→春夏过渡：3月1日~3月15日。

换季时段内展示更强的提醒（换季横幅），过渡期结束后切换为常规季节提示。

### 3.2 待办页 — 季节提示横幅

**位置**：待办页顶部，日期标题下方、任务列表上方。

**展示规则**：

- 换季时段（每年 3月1~15日、9月1~15日）：展示换季提醒横幅（橙色调，更醒目）。
- 非换季时段：展示当季养护小贴士（绿色调，轻量）。
- 用户可关闭横幅（点击"×"），关闭后当季不再展示（下个季节重新出现）。

**横幅内容**：

换季提醒（示例 — 9月初）：
> 🍂 换季提醒：秋冬将至，大多数植物进入休眠期。建议减少浇水频率、停止施肥。
> [查看调整建议]

常规季节提示（示例 — 7月）：
> ☀️ 夏季养护：高温天气植物蒸发加快，注意保持土壤湿润，避免正午强光直射。

**关闭逻辑**：

- 关闭状态存储在 localStorage（key: `seasonal-banner-dismissed-{season}`）。
- 每个季节周期重置（进入新季节时清除上季的关闭状态）。

### 3.3 待办页 — 换季调整建议面板

**触发**：用户点击换季横幅中的"查看调整建议"按钮。

**展示形式**：从底部滑出的 Sheet（半屏）。

**内容**：

- 标题："换季养护调整建议"。
- 列出当前家庭中所有活跃植物（非归档），每盆植物一行：
  - 植物名称 + 当前浇水间隔。
  - 如果已关联品种且品种有季节性建议：展示"建议调整为 X~Y 天"。
  - 如果已使用季节性排期模式：展示"✓ 已自动调整"绿色标签。
  - 如果未关联品种：展示"建议适当调整"通用文案。
- 每行右侧有"去编辑"按钮，跳转到该植物的任务编辑页。
- 底部"知道了"按钮关闭 Sheet。

**不做批量修改**：本次只提供建议和跳转入口，不实现一键批量调整（复杂度高，留给后续）。

### 3.4 植物详情页 — 当季养护要点

**位置**：知识卡片区块内的"当季要点"子区域（依赖 P2-1 植物知识卡片）。

**展示逻辑**：

- 已关联品种：展示品种库中 `seasonalNotes.springSummer` 或 `seasonalNotes.autumnWinter`（根据当前月份）。
- 未关联品种：展示通用季节提示（"春夏生长旺盛期，保持充足水分和光照" / "秋冬休眠期，减少浇水，停止施肥"）。
- 已启用季节性排期的任务：在对应任务旁标注"已自动调整"。

### 3.5 通用季节提示文案库

前端维护一组通用的季节性养护文案（不依赖品种库），用于未关联品种的植物和待办页横幅：

```typescript
const SEASONAL_TIPS = {
  springSummer: {
    banner: "夏季高温，植物蒸发加快，注意保持土壤湿润，避免正午强光直射。",
    detailTip: "春夏生长旺盛期，保持充足水分和光照，可适当施肥促进生长。",
    transitionBanner: "春季来临，植物即将进入生长期。建议逐步增加浇水频率，恢复施肥。",
  },
  autumnWinter: {
    banner: "秋冬季节，植物生长放缓，注意减少浇水避免积水烂根。",
    detailTip: "秋冬休眠期，减少浇水频率，停止或减少施肥，注意保温防冻。",
    transitionBanner: "秋冬将至，大多数植物进入休眠期。建议减少浇水频率、停止施肥。",
  },
};
```

## 4. 数据模型变更

### 4.1 无 schema 变更

本功能为纯前端展示层实现，不需要修改任何 Convex 表结构。所有季节判断和文案展示均在前端完成。

### 4.2 本地存储

使用 localStorage 存储横幅关闭状态：

- Key: `seasonal-banner-dismissed-{springSummer|autumnWinter}`
- Value: `"true"`
- 清理时机：进入新季节时，前端检测并清除上季的 key。

---

## 5. 接口变更

### 5.1 无后端接口变更

本功能不需要任何新的 Convex function 或现有接口修改。

### 5.2 前端需要的数据

待办页的换季调整建议面板需要以下已有数据：

- 家庭植物列表（已有 `plants.listByFamily` query）。
- 每盆植物的任务配置（已有 `plantTasks.listByPlant` query）。
- 植物的 `speciesId`（P2-1 新增字段）。

无需新增查询接口。

## 6. 前端交互设计

### 6.1 待办页季节横幅

- 位置：日期标题（"今天 · 6月23日"）下方，任务分组上方。
- 样式：圆角卡片，左侧 emoji 图标，右侧"×"关闭按钮。
  - 换季提醒：背景 `--color-amber-50`，边框 `--color-amber-200`。
  - 常规提示：背景 `--color-leaf-light`，边框 `--color-leaf` 10% opacity。
- 文案单行或两行，字号 13px，颜色 `--color-ink-light`。
- 换季提醒底部有"查看调整建议"文字按钮（`--color-leaf` 色）。
- 关闭按钮点击后横幅以 fade-out 动画消失（200ms）。

### 6.2 换季调整建议 Sheet

- 从底部滑出，高度约 60% 屏幕。
- 标题区：emoji + "换季养护调整建议" + 副标题"以下植物可能需要调整养护频率"。
- 列表区：每行一盆植物，左侧植物名 + 当前间隔，右侧建议文案或状态标签。
  - "建议调整为 3~5 天"：`--color-amber` 色文字。
  - "✓ 已自动调整"：`--color-leaf` 色标签。
  - "建议适当调整"：`--color-ink-light` 色文字。
- 每行可点击，跳转到对应植物的任务编辑页。
- 底部固定"知道了"按钮，关闭 Sheet。

### 6.3 详情页当季要点

- 集成在知识卡片展开内容的最后一个区块。
- 图标：🌱（春夏）或 🍂（秋冬）。
- 标题："当季要点 · 春夏" 或 "当季要点 · 秋冬"。
- 内容：1~2 行品种级别或通用的季节养护文案。
- 如果该植物有任务使用了季节性排期模式，额外展示一行："系统已按季节自动调整浇水/施肥频率。"

### 6.4 季节切换动效

- 进入新季节时（3月1日/9月1日），如果用户在当天打开应用，横幅以 slide-down 动画出现。
- 横幅内的 emoji 有轻微的 scale 弹跳动画（吸引注意力但不过度）。

## 7. 与其他功能的联动关系

### 7.1 与 P2-1 植物知识卡片的联动

- 季节性建议的品种级文案来源于知识卡片的品种库（`seasonalNotes` 字段）。
- 如果 P2-1 未完成，本功能降级为只展示通用季节提示（不展示品种级建议）。
- 详情页的"当季要点"区块物理上位于知识卡片内部，P2-1 是前置依赖。

### 7.2 与 P1-2 养护任务灵活性的联动

- 已使用季节性排期模式的任务，在换季建议面板中标注"已自动调整"，不再建议手动修改。
- 换季建议面板可引导用户将固定间隔模式切换为季节性模式（"试试季节性排期？"文字链接）。
- 如果 P1-2 未完成，本功能仍可独立工作（只是没有"已自动调整"的标注）。

### 7.3 依赖关系总结

| 依赖 | 类型 | 降级方案 |
|------|------|---------|
| P2-1 植物知识卡片 | 强依赖（详情页当季要点） | 详情页不展示当季要点区块；待办页横幅和换季面板仍可独立工作 |
| P1-2 养护任务灵活性 | 弱依赖（"已自动调整"标注） | 不展示"已自动调整"标签，所有植物都展示调整建议 |

## 8. 边界与异常处理

| 场景 | 处理 |
|------|------|
| 用户在换季时段关闭横幅后又想看 | 本季无法恢复；下个换季时段自动重新出现 |
| 家庭无任何植物 | 不展示换季调整建议面板（无内容可展示）；横幅仍展示通用提示 |
| 所有植物都已使用季节性排期 | 换季面板中所有植物标注"已自动调整"，底部文案改为"所有植物已启用季节性排期，无需手动调整" |
| 用户处于南半球（季节相反） | 本次不处理，按北半球季节展示；后续可根据用户时区推断半球 |
| 月份边界（如 2月28日 vs 3月1日） | 严格按月份判断，2月28日仍为秋冬，3月1日切换为春夏 |
| 品种库中某品种无 seasonalNotes | 该植物详情页展示通用季节提示而非品种级建议 |
| localStorage 被清除 | 横幅重新出现（用户可再次关闭），无功能影响 |

- 一键批量调整所有植物的浇水频率（换季时）

## 11. 设计决策记录

| # | 决策 | 理由 |
|---|------|------|
| D1 | 纯前端实现，无后端变更 | 季节判断和文案展示是确定性逻辑，不需要服务端参与；减少实现复杂度 |
| D2 | 横幅关闭状态存 localStorage 而非用户表 | 轻量级偏好，不值得占用数据库字段；丢失（清缓存）的代价很低（横幅重新出现而已） |
| D3 | 换季时段定义为每月前 15 天 | 给用户足够的缓冲期意识到季节变化；过短（如 3 天）容易错过，过长（如整月）则失去紧迫感 |
| D4 | 不做批量调整，只提供建议和跳转 | 批量修改涉及复杂的确认流程和回滚机制，本次聚焦"提醒"价值；批量操作留给后续迭代 |
| D5 | 季节划分与 P1-2 保持一致（3~8月/9~2月） | 统一心智模型，避免用户在不同功能中看到不同的季节定义 |
| D6 | 南半球暂不处理 | 目标用户群体以中国大陆为主（北半球）；国际化适配留给后续 |
