## 0. 设计基调

本次功能是一个**知识引导层**——它不改变现有核心操作路径（创建植物、查看详情、完成任务），而是在用户"不确定如何养护"的时刻提供恰到好处的建议。

设计性格：**「懂行的朋友在旁边轻声提醒」**——不是弹出式广告（打断流程），不是植物百科词条（信息堆砌），而是融入现有动线的贴心辅助信息。

三条 UI 设计原则：

1. **在场但不打扰**：推荐信息内联展示在用户当前操作区域，不使用模态弹窗，不阻断表单填写流程。
2. **建议而非指令**：视觉上用柔和的辅助色底（`--color-mist`）承载建议，与白色表单区形成对比，明确区分"建议"与"必填操作"。
3. **渐进式披露**：详情页知识卡片默认折叠，不与核心操作区域争夺首屏注意力，用户需要时主动展开。

## 2. 创建植物页 — 品种匹配下拉

### 2.1 设计思路

复用项目中已有的 `LocationAutocomplete` 交互范式——输入框下方弹出候选列表。用户对这种"输入→建议"的模式已有认知，无需额外学习成本。

### 2.2 触发条件

- 用户在"植物名称"输入框中输入 **≥ 2 个字符**后触发品种匹配。
- 匹配结果 > 0 时展示下拉面板。
- 匹配结果 = 0 或输入 < 2 字符时不展示。
- 输入框失焦后 200ms 关闭面板（留出点击候选项的时间）。

### 2.3 整体结构

```

     4px gap
┌────────────────────────────────────────────┐  ← 品种候选面板
│  ┌────────────────────────────────────┐    │
│  │ 🪴  绿萝        容易养护 · 观叶植物 │    │  ← 候选项 1

│  │ 🪴  绿萝吊兰    容易养护 · 观叶植物 │    │  ← 候选项 2
│  ├────────────────────────────────────┤    │
│  │ 🪴  绿宝石      中等难度 · 观叶植物 │    │  ← 候选项 3

└────────────────────────────────────────────┘
```

本节覆盖创建页和编辑页共享的候选列表视觉微交互。

#### 候选项点击反馈

| 阶段 | 时机 | 效果 |
|------|------|------|
| active | 按下瞬间 | 背景色闪烁为 `--color-leaf`（50ms） |
| release | 抬起 | 列表整体开始退场动画 |
| haptic | 按下 + 10ms | 设备振动 10ms（如支持 Vibration API）|

#### 加载态（品种库未就绪）

| 属性 | 值 |
|------|-----|
| 触发条件 | 品种库 JSON 未完成加载且用户已输入 ≥ 2 字符 |
| 延迟展示 | 200ms 后仍未就绪才展示（避免极短加载态闪烁） |
| 骨架屏 | 1 行 shimmer 占位（高度 48px，圆角 8px） |
| shimmer 色 | 从 `--color-mist` 到 `--color-surface` 往返渐变，800ms 循环 |
| 实际概率 | 极低（前端 JSON，首屏加载后几乎不触发） |

#### 空结果态

- 不展示任何下拉内容。
- 无"无结果"文案——静默消失，避免干扰自由输入流。
- 如面板之前可见，以 100ms fade-out 退场。

#### 性能指标（视觉流畅性）

| 指标 | 要求 |
|------|------|
| 列表渲染帧率 | 入场/退场动画 60fps，无掉帧 |
| 首次匹配响应 | 输入停止 → 列表出现 ≤ 250ms（200ms debounce + 50ms 渲染） |
| DOM 节点数 | ≤ 3 条 option DOM（无虚拟滚动需求） |

### 2.10 品种选择确认态（v1.1 新增）

用户点击候选项后，输入框从"普通输入态"切换为"已关联确认态"。此状态在视觉上明确传达"名称字段不再是纯文本，而是已关联结构化品种数据"。

#### 状态切换时序

| 步骤 | 时机 | 视觉变化 |
|------|------|----------|
| 1 | 0ms（点击候选项） | 候选面板开始消失动效 |
| 2 | 0ms | 输入框文本替换为品种标准名（`names[0]`） |
| 3 | 50ms | 输入框样式过渡至确认态（150ms ease-out） |
| 4 | 150ms | 左侧品种图标淡入、右侧清除按钮淡入 |
| 5 | 200ms | 确认态就绪，推荐卡片开始入场动效 |

| 步骤 | 时机 | 效果 |
|------|------|------|
| 1 | 0ms | 清除按钮、品种图标开始退场动画 |
| 2 | 100ms | 图标消失完成 |
| 3 | 100ms | 输入框样式过渡回普通态（背景、边框、padding） |
| 4 | 250ms | 过渡完成，输入框恢复为标准文本输入状态 |

**设计决策**：为什么用输入框整体视觉变化而不是独立的 pill 标签？

在创建页面中，输入框就是用户交互的焦点。如果在输入框之外增加独立标签，用户会困惑"输入框里的文字"和"标签"之间的关系。直接让输入框自身变化（底色、图标、字重），用户一眼就能感知状态切换——"我刚才的选择已生效，这个输入框现在代表一个已确认的品种"。

## 3. 创建植物页 — 推荐卡片

### 3.1 设计思路

推荐卡片是整个功能最关键的 UI 表达。需要同时传达三层信息：①"这是什么品种"（识别）②"系统建议这样养"（建议内容）③"你可以一键应用，也可以忽略"（操作选择）。

采用 inline 卡片而非 Bottom Sheet 的原因：Bottom Sheet 是打断性交互，会遮挡用户正在填写的表单内容；inline 卡片让用户在不离开当前视觉焦点的情况下获取建议并做出选择。

### 3.2 位置与触发

- **触发条件**：用户点击品种候选列表中的某一项。
- **展示位置**：植物名称输入框所在卡片组的下方，第二个表单卡片组（简介）的上方。
- **与周围元素的间距**：与上方名称卡片 `var(--space-sm)`（8px），与下方简介卡片 `var(--space-md)`（16px）。

### 3.3 整体结构

```
│  └─────────────────────────────────────────────────────┘  │
```

| 状态 | 视觉变化 |
|------|---------|
| 默认 | 如上规格 |
| hover/active | 背景变为 `--color-leaf-light`，`transform: scale(0.97)` |
| 松开 | 回弹 `scale(1)`，150ms ease-out |
| disabled（提交中） | `opacity: 0.6`，cursor: not-allowed |

**设计决策**：为什么用 `--color-leaf` 而不是 `--color-gold`？

金色（CTA）在本系统中代表"主动操作"（保存、添加）——用户发起的意图。"应用推荐"更像是"接受系统建议"——一个相对被动的确认动作。深绿色让它在视觉上温和，不会像金色那样激进地呼唤用户点击，尊重"建议而非指令"的设计原则。

| 2 | 表单数据加载完成 | 输入框文本填入植物名称 |
| 3 | +50ms | 输入框过渡至确认态（底色 + 左侧图标 + 右侧 ×） |
| 4 | +100ms | 品种关联标签（§5.2）入场动画 |

**取消关联 Toast 规格**：

| 属性 | 值 |
|------|-----|
| 触发 | 用户点击确认态 × 按钮或品种标签 × 按钮 |
| 文案 | `"已取消品种关联"` |
| icon | 无 emoji（中性操作） |
| 停留时长 | 2s（比"应用推荐"的 3s 更短——轻量操作轻量反馈） |
| 样式 | 复用 GlobalToast 规格（见 §3.10） |

### 5.6 编辑页重新匹配的视觉流程（v1.1 新增）

当用户取消关联后重新输入文字触发品种匹配：

```
┌─ 时间线 ────────────────────────────────────────────────────────┐
│                                                                  │
│  [确认态]  → 点击× → [退场动画 250ms] → [普通态]               │
│                                                                  │
│  [普通态]  → 输入≥2字符 → [debounce 200ms] → [候选列表入场]     │
│                                                                  │
│  [候选列表] → 点击候选项 → [确认态入场 150ms] + [标签入场]      │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

**注意**：编辑页不展示推荐卡片，因此"选择候选项 → 确认态"是终点动作。输入框进入确认态后用户的下一步是"保存"或"继续编辑其他字段"。

## 14. 边界与异常处理（视觉维度）

| 场景 | 视觉处理 |
|------|---------|
| 品种名过长（超过单行宽度） | 候选项品种名 `text-overflow: ellipsis`，单行截断 |
| 三宫格贴士文字过长 | 单行 ellipsis |
| 知识区块内容过长 | 不截断，完整展示（由数据源保证合理长度） |
| 常见问题列表过多（>5条） | 全部展示，不做折叠（数据源保证 ≤ 5 条） |
| 推荐卡片三宫格某项缺数据 | 该格位不展示（flex 自动填满剩余空间） |
| 名称输入后快速删除（debounce） | 匹配逻辑 debounce 200ms，避免频繁闪烁 |
| 品种库 JSON 未加载完成 | 输入时不触发匹配，不展示面板（静默等待） |
| 确认态品种名超长（>15字） | 输入框内文本 ellipsis，左侧图标和右侧 × 按钮不被挤压（flex-shrink: 0） |
| 确认态下用户粘贴新文本 | 等同于修改文本：即时退出确认态，清除 speciesId，重新匹配 |
| 快速连续点击候选项（重复点击） | 以最后一次为准，中间态不残留（speciesId 取最后一次选择值） |
| 预填简介/备注时字段已有内容 | 跳过该字段预填，不参与高亮动效，Toast 切换为"部分跳过"文案 |
| 输入超长文本（> 50 字符） | 不触发匹配，输入框保持普通态 |
| 候选项点击后推荐卡片未渲染完成即退场 | 确认态 + 推荐卡片的动画互不阻塞，分别独立完成 |
| 编辑页取消关联后立即保存 | speciesId 标记为 null，确认态已退出，保存按钮正常可用 |

## 16. 设计决策记录

| # | 决策 | 理由 |
|---|------|------|
| UD1 | 推荐卡片用 inline 展示而非 Bottom Sheet | Sheet 是打断性交互，会遮挡表单；inline 卡片让用户在当前视觉焦点内获取信息 |
| UD2 | 详情页知识卡片默认折叠 | 详情页核心价值是操作工作台，知识是辅助信息；默认折叠避免信息过载 |
| UD3 | 三宫格横排而非竖向列表 | 紧凑性：推荐卡片需控制高度；语义：三维度是并列关系，水平排列传达等权 |
| UD4 | "应用推荐"用 `--color-leaf` 而非 `--color-gold` | 区分"接受建议"（被动确认）与"保存/添加"（主动操作）的语义层级 |
| UD5 | "仅供参考"用文字链而非按钮 | 拒绝动作不该与主路径视觉等权；文字链让主按钮成为唯一焦点 |
| UD6 | 知识区块用 `--color-mist` 底色 | 在白色容器内形成信息分组层次；复用系统中"辅助信息"的色彩语义 |
| UD7 | 复用 `LocationAutocomplete` 交互范式 | 保持用户认知一致性，降低学习成本；代码层面可复用匹配逻辑 |
| UD8 | 编辑页不展示推荐卡片 | 植物已创建，任务参数已存在；展示推荐卡片可能误导用户覆盖已有设置 |
| UD9 | Chevron 旋转而非文字切换（▾/▴） | 旋转动画更流畅，是移动端折叠组件的通用惯例 |
| UD10 | 候选面板最多 3 条 | 避免面板过高遮挡表单内容；3 条足够覆盖常见匹配场景 |
| UD11 | 确认态用输入框整体视觉变化而非独立 pill 标签（创建页） | 输入框是用户唯一焦点，整体变色比外挂标签更直觉；减少 DOM 层级，避免标签与输入框的关系困惑 |
| UD12 | 确认态下边框 2px 而非 1px | 增加视觉分量感，让确认态和普通态即使在快速扫视下也能被区分 |
| UD13 | 清除按钮 active 态用 `--color-error` 而非 `--color-ink` | "取消关联"是一个轻量破坏性操作，红色瞬间暗示"正在移除某个绑定"，但因是 active 态不持续显示所以不会造成焦虑 |
| UD14 | 预填字段 stagger 间隔 80ms 而非同时高亮 | stagger 动画引导用户视线依次扫过被填充的字段，形成"推荐正在被逐步应用"的感知节奏；同时闪烁会让用户不知道该看哪里 |
| UD15 | Toast 停留时长区分（应用推荐 3s / 取消关联 2s） | 操作影响范围大（多字段预填）给更多阅读时间；轻量操作（取消）快速确认即可 |
| UD16 | 编辑页复用确认态但附加独立标签 | 编辑页承载更复杂的信息密度，确认态 + 独立标签双重信号避免用户"不确定当前植物绑的是什么品种" |
| UD17 | 确认态品种图标用 emoji 🌿 而非 SVG | 与推荐卡片头部保持视觉一致；emoji 在小尺寸（18px）下辨识度反而优于线条 SVG |
