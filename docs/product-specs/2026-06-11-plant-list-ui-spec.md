## 0. 设计基调

植物列表页是**中频浏览页**，UI 必须服务 PRD 的三个核心价值：**资产总览**、**快速识别**、**状态感知**。

由此确立本页的视觉性格：**「安静的植物档案柜」**——不是花哨的相册瀑布流，不是密集的后台管理表格，而是像一本翻开的家庭植物手账：底色温润（`--color-paper`）、卡片干净（`--color-surface`）、图片是识别锚点而非装饰主角、状态信息用色彩而非文字量传达紧迫感。

三条 UI 设计原则：

1. **密度适中，扫描优先**：一屏至少 4 株植物完整可见，用户无需逐个细看即可感知整体状态。
2. **图片服务识别，不服务展示**：80×80pt 方图足够认出「这是哪盆」，把空间让给状态信息。
3. **状态由色彩承载，层级由排版建立**：逾期用暖橙色条，正常用灰色辅助文字，名称永远是视觉第一层级。

## 1. 整体布局结构

植物列表页自上而下分四层，垂直堆叠，移动端单列：

```
┌─────────────────────────────────────┐
│  ① 标题栏 (Title Bar)                │  ← 页面标题 + 添加植物按钮
├─────────────────────────────────────┤
│  ② 搜索栏 (Search)                   │  ← 辅助查找，视觉轻量
├─────────────────────────────────────┤
│  ③ 活跃植物列表 (Active List)          │  ← 主区，横向紧凑卡片
│     · 卡片 · 卡片 · 卡片 …            │
├─────────────────────────────────────┤
│  ④ 归档折叠区 (Archived)              │  ← 底部折叠，默认收起
│     ▶ 已归档 (N)                      │
└─────────────────────────────────────┘
          [ 固定底部导航 ]                ← 三 Tab：待办 / 植物 / 设置
```

布局要点：

- **标题栏与搜索栏之间用 `--space-md`（16px）间距分隔**，不用分割线。
- **活跃列表区是页面主体**，占据绝大部分视口高度，卡片之间用 `--space-sm`（8px）间距。间距小于待办页卡片间距（待办页 12px），因为植物卡片本身更紧凑，需要更密的节奏感。
- **归档区与活跃列表之间用 `--space-lg`（24px）+ 一条 1px `--color-line` 分割线**，明确区分「日常」与「存档」。
- 页面底部预留 `80px` 安全区（底部导航高度 + 底部安全距离）。

## 2. ① 标题栏设计

### 2.1 结构

```
┌─────────────────────────────────────┐
│  家庭植物档案              [ + 添加植物 ] │
└─────────────────────────────────────┘
```

### 2.2 视觉规格

| 元素 | 规格 |
|------|------|
| 标题文字 | `--font-heading`（Lora），24px，font-weight 700，`--color-ink` |
| 标题行高 | 1.2 |
| 添加按钮 | 文字型按钮，14px，`--color-leaf`，无填充背景 |
| 按钮前缀 | `+` 符号，与文字同色同大小 |
| 整行高度 | 48px（含上下 padding） |
| 对齐方式 | 左右两端对齐（`justify-content: space-between`） |

### 2.3 设计意图

- 标题用 Lora 衬线体保持「手账」气质，但尺寸收敛（24px 而非 32px），不抢卡片区注意力。
- 添加按钮不用填充态主按钮（避免与卡片区争夺视觉焦点），用文字链形态，轻量但可发现。
- 当列表为空时，添加按钮仍在标题栏保留，空状态区域另有独立 CTA。

## 3. ② 搜索栏设计

### 3.1 视觉规格

| 属性 | 值 |
|------|-----|
| 高度 | 40px |
| 背景 | `--color-mist` |
| 边框 | 无（靠背景色与页面底色区分） |
| 圆角 | `--radius-input`（12px） |
| 内边距 | 0 12px |
| 字体 | `--font-body`，14px，`--color-ink` |
| placeholder | 「按名称或位置搜索」，`--color-muted` |
| 左侧图标 | 🔍 或 SVG 搜索图标，16px，`--color-muted`，可选 |

### 3.2 交互状态

| 状态 | 视觉变化 |
|------|---------|
| 默认 | `--color-mist` 底色，无边框 |
| 聚焦 | 底色不变，出现 2px `--color-leaf` 内描边（`box-shadow: inset 0 0 0 2px`） |
| 有输入内容 | 右侧出现清除按钮（× 图标，`--color-muted`，点击清空） |

### 3.3 设计意图

- 去掉当前的 1px border，改用纯背景色区分。搜索栏是辅助工具，不应有「表单输入框」的正式感。
- 聚焦时用主色内描边而非外发光，保持安静。

## 4. ③ 植物卡片设计（核心）

### 4.1 整体结构

采用 PRD 确定的**横向紧凑布局**（左图右文），整卡可点击：

```
┌──────────────────────────────────────────────────┐
│ ┌────────┐                                        │
│ │        │  发财树                          ✏️    │
│ │  图片   │  📍 客厅茶几                          │
│ │ 80×80  │  💧 浇水 · 明天到期                    │
│ │        │                                        │
│ └────────┘                                        │
└──────────────────────────────────────────────────┘
```

### 4.2 卡片容器规格

| 属性 | 值 | 说明 |
|------|-----|------|
| 布局 | `display: flex; align-items: center` | 横向排列，垂直居中 |
| 内边距 | 12px | 四周等距 |
| 背景 | `--color-surface` | 白色卡片 |
| 圆角 | `--radius-card`（16px） | 与全局卡片圆角一致 |
| 边框 | 1px solid `--color-line` | 轻量边界感 |
| 阴影 | 无（默认态） | 遵循「间距 > 分组 > 阴影」优先级 |
| 最小高度 | 104px | 保证图片 + padding 的最小空间 |
| 间距（卡片之间） | `--space-sm`（8px） | 紧凑节奏 |

### 4.3 图片区规格

| 属性 | 值 | 说明 |
|------|-----|------|
| 尺寸 | 80×80pt | 固定正方形 |
| 圆角 | 12px | 略小于卡片圆角，形成内嵌感 |
| object-fit | cover | 裁切填充，不留白边 |
| flex-shrink | 0 | 不被压缩 |
| 右侧间距 | `--space-md`（16px） | 图片与文字区的间隔 |

**无图占位态**：

```
┌────────┐
│        │
│   🌿   │  ← 32px emoji，居中
│        │
└────────┘
```

- 背景：`--color-mist`
- emoji 颜色：自然色（不做灰度处理）
- 圆角、尺寸与有图态完全一致

**图片加载态**：

- 初始：`--color-mist` 纯色块（与占位态视觉一致）
- 加载完成：`opacity 0 → 1`，200ms ease-in，图片渐显覆盖色块

### 4.4 文字区规格

文字区为右侧弹性区域（`flex: 1; min-width: 0`），内部垂直堆叠三行信息：

```
┌──────────────────────────────────┐
│  发财树                    ✏️    │  ← 第一行：名称 + 编辑图标
│  📍 客厅茶几                     │  ← 第二行：位置（可选）
│  💧 浇水 · 明天到期              │  ← 第三行：任务状态
└──────────────────────────────────┘
```

**第一行：植物名称 + 编辑图标**

| 元素 | 规格 |
|------|------|
| 名称字体 | `--font-heading`（Lora），16px，font-weight 600 |
| 名称颜色 | `--color-ink` |
| 名称行高 | 1.3 |
| 名称溢出 | 单行，`text-overflow: ellipsis` |
| 编辑图标 | 铅笔 SVG，16×16，`--color-muted`，opacity 0.5 |
| 编辑图标位置 | 行尾，`margin-left: auto` |
| 编辑图标热区 | 44×44（padding 扩展），`position: relative` |

**第二行：摆放位置**

| 元素 | 规格 |
|------|------|
| 前缀 | 📍 emoji，12px |
| 文字 | `--font-body`，12px，`--color-muted`，font-weight 400 |
| 行高 | 1.4 |
| 与第一行间距 | 2px |
| 条件 | 仅当 `location` 有值时展示，否则此行不渲染 |

**第三行：任务状态行**

| 元素 | 规格 |
|------|------|
| 前缀 | 任务类型 emoji，12px |
| 文字 | `--font-body`，12px，font-weight 500 |
| 分隔符 | ` · `（中点，两侧各一空格） |
| 与第二行间距 | 4px |
| 条件 | 有任务时展示类型 + 到期文案；无任务时展示「还没有养护任务」 |

### 4.5 任务状态行的色彩规则

状态行的颜色随到期紧迫度变化，这是卡片中**唯一的动态色彩信号**：

| 状态 | 文字颜色 | 附加视觉 | 设计意图 |
|------|---------|---------|---------|
| 已逾期 | `--color-warning` | 卡片左侧 3px 暖橙色条（`border-left`） | 温和警示，不施压 |
| 今天到期 | `--color-ink` | 无附加 | 正常优先级，不额外强调 |
| 明天/后天到期 | `--color-muted` | 无附加 | 预期信息，弱化 |
| 3+ 天后到期 | `--color-muted` | 无附加 | 最弱优先级 |
| 无任务 | `--color-muted`，opacity 0.7 | 无附加 | 最弱，暗示需要配置 |

**逾期色条细节**：

- 位置：卡片最左侧，从上到下贯穿
- 宽度：3px
- 颜色：`--color-warning`
- 圆角：左侧跟随卡片圆角（`border-radius: 16px 0 0 16px` 的左侧部分自然形成）
- 实现：`border-left: 3px solid var(--color-warning)`，卡片 padding-left 相应减少 3px

### 4.6 编辑图标交互

| 状态 | 视觉 |
|------|------|
| 默认 | opacity 0.4，`--color-muted` |
| 卡片被按下时 | 跟随卡片缩放，不单独变化 |
| 编辑图标被按下时 | opacity 1.0，过渡 100ms |
| 编辑图标松开 | 跳转编辑页 |

编辑图标必须阻止事件冒泡（`e.stopPropagation()`），避免同时触发卡片的详情跳转。

## 5. ④ 归档折叠区设计

### 5.1 折叠触发器

```
┌─────────────────────────────────────┐
│  ▶ 已归档 (3)                        │  ← 收起态
└─────────────────────────────────────┘
```

| 属性 | 值 |
|------|-----|
| 高度 | 44px |
| 字体 | `--font-body`，13px，font-weight 500 |
| 颜色 | `--color-muted` |
| 箭头 | ▶（收起）/ ▼（展开），与文字同色 |
| 背景 | 透明 |
| 点击区域 | 整行可点击 |
| 上方分隔 | 1px `--color-line` + 24px 上间距 |

### 5.2 归档卡片（展开后）

归档卡片与活跃卡片结构相同，但做**灰态处理**：

| 属性 | 与活跃卡片的差异 |
|------|----------------|
| 整体 opacity | 0.55 |
| 图片 | 加 `filter: grayscale(0.6)` |
| 名称颜色 | `--color-muted`（而非 `--color-ink`） |
| 第二行内容 | 显示归档时间「2026-05-20 归档」替代位置 |
| 第三行内容 | 不显示（归档后任务无意义） |
| 编辑图标 | 不显示 |
| 右侧操作 | 显示「恢复」文字按钮，`--color-leaf`，14px |
| 点击行为 | 整卡不可点击进详情（需先恢复） |
| 左侧色条 | 无（归档无紧迫状态） |

### 5.3 展开/收起动效

- 展开：内容区 `max-height: 0 → auto`，300ms ease-out + `opacity 0 → 1`
- 收起：反向，200ms ease-in
- 箭头旋转：▶ → ▼，`transform: rotate(90deg)`，200ms

## 6. 卡片按压与点击动效

### 6.1 按压反馈（Active State）

| 属性 | 值 |
|------|-----|
| 触发 | `:active` 或 touch start |
| 变换 | `transform: scale(0.98)` |
| 过渡（按下） | 120ms `ease-out` |
| 过渡（松开） | 200ms `cubic-bezier(0.34, 1.56, 0.64, 1)`（轻微回弹） |
| 背景变化 | `--color-surface` → `--color-mist`（极轻微底色变化） |

### 6.2 列表渐入动效（Stagger Fade-in）

页面首次加载数据后，卡片列表执行一次入场动画：

| 属性 | 值 |
|------|-----|
| 初始状态 | `opacity: 0; transform: translateY(8px)` |
| 结束状态 | `opacity: 1; transform: translateY(0)` |
| 单卡时长 | 250ms |
| 缓动 | `cubic-bezier(0.25, 0.46, 0.45, 0.94)` |
| 卡片间延迟 | 50ms（stagger） |
| 总时长上限 | 前 6 张卡片有 stagger，第 7 张起同时出现（避免长列表等待过久） |
| reduced-motion | 跳过动画，直接显示 |

### 6.3 图片渐显

| 属性 | 值 |
|------|-----|
| 初始 | `opacity: 0`（图片 DOM 已存在，背景为 `--color-mist`） |
| 加载完成 | `opacity: 1`，200ms ease-in |
| 失败 | 保持占位态（🌿），不闪烁 |

## 8. 色彩应用总览

本页面涉及的色彩 token 使用场景汇总：

| Token | 本页使用场景 |
|-------|------------|
| `--color-paper` | 页面背景 |
| `--color-surface` | 卡片背景 |
| `--color-mist` | 搜索栏背景、图片占位背景、卡片按压态背景 |
| `--color-ink` | 标题、植物名称、今天到期状态文字 |
| `--color-muted` | 位置文字、未来到期状态文字、编辑图标、归档区文字 |
| `--color-leaf` | 搜索栏聚焦描边、添加按钮文字、恢复按钮文字 |
| `--color-gold` | 空状态 CTA 按钮填充 |
| `--color-warning` | 逾期状态文字、逾期卡片左侧色条 |
| `--color-line` | 卡片边框、归档区上方分割线 |

**色彩约束**：

- 本页面不出现 `--color-error`（无危险操作）
- 本页面不出现 `--color-success`（无完成动作）
- `--color-gold` 仅在空状态 CTA 出现一次，正常列表态无强饱和色块
- 逾期色条是页面中唯一的「彩色信号」，确保其视觉突出性

## 11. 无障碍要求

| 要求 | 实现方式 |
|------|---------|
| 卡片可聚焦 | `tabIndex={0}`，`role="article"` 或语义化 `<article>` |
| 卡片 accessible name | `aria-label="发财树，客厅茶几，浇水明天到期"` |
| 编辑按钮 accessible name | `aria-label="编辑发财树"` |
| 状态不只靠颜色 | 逾期同时有文案「逾期 N 天」+ 色条双编码 |
| 归档折叠 | `aria-expanded="true/false"`，`aria-controls` 指向内容区 |
| 图片 alt | `alt="发财树封面图"` 或占位态 `alt=""` + `aria-hidden` |
| 对比度 | 所有文字对比度 ≥ 4.5:1（`--color-muted` #637b71 on `--color-surface` #fff = 4.6:1 ✓） |

## 12. 与待办页的视觉一致性

植物列表页与待办页同属认证后主流程页面，需保持以下一致性：

| 维度 | 统一规则 |
|------|---------|
| 页面背景 | 均为 `--color-paper` |
| 标题风格 | 均为 Lora 24px 700，左对齐 |
| 卡片圆角 | 均为 `--radius-card`（16px） |
| 卡片边框 | 均为 1px `--color-line` |
| 逾期色彩 | 均为 `--color-warning`，不用红色 |
| 动效节制 | 均只做渐入 + 按压反馈，不做装饰性动画 |
| 底部导航 | 共享同一组件，当前页高亮 |

**差异点**（有意为之）：

| 维度 | 待办页 | 植物列表页 | 理由 |
|------|--------|-----------|------|
| 卡片间距 | 12px | 8px | 植物卡片更紧凑，需要更密的节奏 |
| 卡片阴影 | `--shadow-card` | 无 | 待办卡片需要「浮起」感（可操作），植物卡片是平铺浏览 |
| 情感反馈区 | 有（Greeting Card） | 无 | 植物列表不需要情感激励，纯功能 |
| 主色块 | `--color-gold`（完成按钮） | 无（正常态无强色块） | 植物列表无高频 CTA |
